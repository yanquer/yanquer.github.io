MacOS下文件IO MacOS下Flutter启动会先黑屏几秒

No Material widget found.

出现原因

使用了 Material 风格的 widget 就需要 Scaffold 作为根布局

Material 风格的 widget

  • AppBar
  • BottomAppBar
  • FloatingActionButton
  • Drawer
  • SnackBar
  • BottomSheet
  • BottomNavigationBar
  • ListTile
  • IconButton
  • 等

解决就是当要使用这些控件时, 外部用 Scaffold 包裹

// 仅贴出关键部分

Widget build(BuildContext context) {
  return Scaffold(
    body: ...,
  )
}

参考: https://blog.csdn.net/yechaoa/article/details/90693377

A RenderFlex overflowed by 25447 pixels on the bottom.

当 Column 下有 ListView 这种不确定高度的组件时, 可能就会遇到这个报错, 或者是这个报错:

Cannot hit test a render box with no size.

如果直接指定 Column 高度, 无法确定 ListView 能够填充满剩余空间.

这个时候可以考虑使用 Expanded 组件

@override
Widget build(BuildContext context) {

  return Column(
    children: [
      // 搜索框

      // 总数
      SizedBox(height: 10,),
      Text("共 $count 首音乐"),

      Expanded(child: ListView(
        shrinkWrap: true,
        children: [
          ...musicElement,
        ],
      ))
    ],
  );
}
