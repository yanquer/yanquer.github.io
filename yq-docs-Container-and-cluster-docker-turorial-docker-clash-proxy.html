docker容器使用clash代理

修改于: 2025-04-12

单个容器内使用clash代理

实际上给容器设置以下环境变量即可

  • http_proxy
  • https_proxy
  • no_proxy

大小写不敏感

注解

clash方的配置此处省略, 部分可参考 :clash等代理问题

cli设置

docker run --env http_proxy=http://192.168.1.2:7890 --env https_proxy=http://192.168.1.2:7890 --env no_proxy=localhost,192.168.1.2 nginx:latest

注解

此处假定你的代理地址是 192.168.1.2

compose设置

services:
  nginx:
    image: nginx:latest
    environment:
      - http_proxy=http://192.168.1.2:7890
      - https_proxy=http://192.168.1.2:7890
      - no_proxy=localhost,192.168.1.2

如果你的clash配置了登陆校验, 则需要在 http_proxy 的路径前面加上 http://username:password@, 比如 http://192.168.1.2:7890 变成 http://username:password@192.168.1.2:7890

docker引擎全局使用clash代理

关键字
  • docker pull使用代理

使用此方案有个好处, 可以直接 pull 到想要的 image 不需要额外从其他地方弄

步骤

创建 systemd 配置目录(存在则跳过)

sudo mkdir -p /etc/systemd/system/docker.service.d

创建代理配置文件

sudo touch /etc/systemd/system/docker.service.d/http-proxy.conf

编辑配置文件:

[Service]
  Environment="HTTP_PROXY=http://192.168.1.2:7890/"
  Environment="HTTPS_PROXY=http://192.168.1.2:7890/"
  Environment="NO_PROXY=localhost,127.0.0.1,.example.com"

如果有密码认证就是:

[Service]
  Environment="HTTP_PROXY=http://username:password@192.168.1.2:7890/"
  Environment="HTTPS_PROXY=http://username:password@192.168.1.2:7890/"
  Environment="NO_PROXY=localhost,127.0.0.1"

重新加载 systemd 配置并重启 Docker

sudo systemctl daemon-reload
sudo systemctl restart docker

验证配置是否生效

# 能看到代理的详细账密
sudo systemctl show --property=Environment docker

# 或者
sudo docker info | grep -A 1 "HTTP Proxy"
# 输出
# HTTP Proxy: http://xxxxx:xxxxx@192.168.1.2:7890
# HTTPS Proxy: http://xxxxx:xxxxx@192.168.1.2:7890
# 注意有账户密码的时候, 这部分是会被其他符号替换的, 是正常的
