节流与消抖

节流(throttle): n 秒内只运行一次，若在 n 秒内重复触发，只有一次生效; 在一定时间内，事件最多触发一次。即使事件频繁触发，也只在规定的时间间隔内执行一次。

节流 关注的是固定时间间隔内的执行次数，确保事件在指定时间内只执行一次。

消抖(debounce): n 秒后在执行该事件，若在 n 秒内被重复触发，则重新计时; 在事件触发后，等待一段时间，若在这段时间内没有再次触发事件，才执行操作。如果事件再次触发，则重新计时。

消抖动 关注的是事件触发后的等待时间，确保在事件停止触发后才执行操作。

参考: https://vue3js.cn/interview/JavaScript/debounce_throttle.html#代码实现

适用场景
节流
  • 滚动事件（scroll）：比如无限加载列表，每隔一段时间检查一次滚动位置。
  • 窗口调整（resize）：每隔一段时间调整一次布局。
  • 鼠标移动（mousemove）：比如拖拽操作，限制触发频率。
消抖
  • 搜索框输入（input）：等待用户停止输入后再发送请求。
  • 按钮点击（click）：防止用户多次点击导致重复提交。
  • 窗口调整（resize）：等待用户停止调整窗口后再执行操作。

重要

此库提供的节流与消抖, 是一个回调而不是函数调用.

类似于:

const Sleep = (ms: number)=> {
  return new Promise(resolve=>setTimeout(resolve, ms))
}

const debounce = async (func, wait: number) => {
  if (typeof func !== 'function') {
    throw new TypeError('Expected a function')
  }

  // 等待 xxx ms
  await Sleep(wait);
  return func
}

这里的代码其实有点问题, 没有实现到 重复触发，重新计时

lodash用于节流(throttle)函数的调用频率,以提高性能。

限制指定的时间间隔内最多执行一次原函数。也就是说,它限制了调用函数的频率,避免函数被过于频繁地调用。

使用 lodash.throttle 的好处有:

  • 防止频繁的事件触发导致性能问题,如 resize、scroll 等事件的处理函数。
  • 控制动画或轮询功能的频率,避免过度占用 CPU 资源。
  • 网络请求频率控制,减少不必要的请求量。
  • 输入框实时搜索suggestions时限制请求频率。

例如:

import { throttle } from 'lodash';

function handleResize() {
  // 处理窗口 resize
}

window.addEventListener('resize', throttle(handleResize, 100));

这将限制 handleResize 函数每 100ms 至多执行一次,从而提高页面滚动时的性能。

节流/消抖失效

常见于React的函数组件.

如果函数组件内部使用了节流/消抖函数, 那么当组件重新渲染时, 节流/消抖函数的回调函数也会重新创建, 这会导致节流/消抖失效。

比如

export const MyComponent = () => {

  const handleChange = throttle((val: string) => {
    console.log(val);
  }, 1000);

  return (
    <input
      type="text"
      onChange={event => handleChange(event.target.value)}
    ></input>
  )

}

如果 MyComponent 组件重新渲染, 那么 handleChange 函数也会重新创建, 新的节流函数与之前的无关, 节流失效。

解决方法

可以使用

  • useCallback
  • useMemo
  • useRef

来保存节流/消抖函数, 从而避免函数重新创建。

export default function MyComponent() {
  const [count, setCount] = useState(0);

  const handleChange = useCallback(
    throttle((val: string) => {
      console.log(val);
    }, 1000),

    []);

  return (
    <input
      type="text"
      onChange={event => handleChange(event.target.value)}
    ></input>
  )
}

注解

用例是手写的伪代码, 不一定能直接运行...
