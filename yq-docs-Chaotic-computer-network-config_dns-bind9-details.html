BIND9配置项详细说明

修改于: 2025-04-12

主配置文件结构

主要配置文件通常位于 /etc/bind/

  • named.conf - 主配置文件
  • named.conf.options - 全局选项
  • named.conf.local - 本地区域定义
  • named.conf.default-zones - 默认区域(如localhost、根提示等)

named.conf.options

最重要的全局选项配置文件

options {
    // 基本设置
    directory "/var/cache/bind";    # 工作目录
    pid-file "/var/run/named/named.pid"; # PID文件位置
    statistics-file "/var/log/named.stats"; # 统计文件

    // 监听设置
    listen-on { any; };             # 监听所有IPv4接口
    listen-on-v6 { any; };          # 监听所有IPv6接口

    // 访问控制
    allow-query { any; };           # 允许查询的客户端
    allow-recursion { 192.168.1.0/24; }; # 允许递归查询的客户端
    allow-transfer { none; };       # 限制区域传输

    // 转发设置
    forwarders {                    # 设置上游DNS
        8.8.8.8;
        8.8.4.4;
    };
    forward only;                   # 仅使用转发器

    // DNSSEC设置
    dnssec-validation auto;         # 自动DNSSEC验证
    dnssec-enable yes;              # 启用DNSSEC支持

    // 缓存设置
    max-cache-size 256M;            # 最大缓存大小
    max-cache-ttl 3600;             # 最大缓存TTL

    // 日志设置
    version "not currently available"; # 隐藏BIND版本
};

named.conf.local

定义本地区域和视图：

// 主区域定义
zone "example.com" {
    type master;                    # 主服务器
    file "/etc/bind/db.example.com"; # 区域文件路径
    allow-update { key "rndc-key"; }; # 动态更新控制
    allow-transfer { 192.168.1.2; }; # 从服务器IP
    notify yes;                     # 启用区域变更通知
};

// 从区域定义
zone "example.org" {
    type slave;                     # 从服务器
    file "slave/db.example.org";    # 区域文件保存位置
    masters { 192.168.1.1; };      # 主服务器IP
};

// 反向解析区域
zone "1.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192.168.1";
};

// 根提示区域(必须)
zone "." {
    type hint;
    file "/etc/bind/db.root";
};

区域文件格式说明

区域文件(如db.example.com)包含资源记录：

$TTL 86400                      # 默认TTL值
@       IN SOA  ns1.example.com. admin.example.com. (
                2024041801      ; 序列号
                3600            ; 刷新时间(秒)
                900             ; 重试间隔
                604800          ; 过期时间
                86400 )         ; 最小TTL

; 名称服务器记录, 即nameserver
        IN NS   ns1.example.com.
        IN NS   ns2.example.com.

; 邮件交换记录
        IN MX 10 mail.example.com.

; A 记录(主机记录)
ns1     IN A    192.168.1.1
ns2     IN A    192.168.1.2
www     IN A    192.168.1.10
mail    IN A    192.168.1.20

; CNAME 记录(别名)
web     IN CNAME www

; TXT记录
        IN TXT  "v=spf1 mx -all"

; SRV记录
_sip._tcp IN SRV 10 60 5060 sipserver

反向区域文件

db.192.168.1 文件内容：

$TTL 86400
@ IN SOA ns1.example.com. admin.example.com. (
            2024041801
            3600
            900
            604800
            86400 )

      IN NS   ns1.example.com.
      IN NS   ns2.example.com.

1      IN PTR  ns1.example.com.
2      IN PTR  ns2.example.com.
10     IN PTR  www.example.com.
20     IN PTR  mail.example.com.

关键配置指令

allow-query
允许查询的客户端 { 192.168.1.0/24; }
allow-recursion
允许递归查询的客户端 { localhost; 192.168.1.0/24; }
allow-transfer
允许区域传输的服务器 { 192.168.1.2; }
allow-update
允许动态更新的客户端 { key "rndc-key"; }
notify
是否通知从服务器, yes 或 no
also-notify
额外通知的服务器 { 192.168.1.3; }
forwarders
上游DNS服务器 { 8.8.8.8; 1.1.1.1; }
forward

转发模式

  • only(仅转发)
  • first(先尝试转发)

安全最佳实践

限制访问：

allow-query { localhost; 192.168.1.0/24; };
allow-recursion { localhost; 192.168.1.0/24; };
allow-transfer { none; };  # 或仅限从服务器

隐藏版本信息：

version "not disclosed";

使用TSIG密钥认证：

key "rndc-key" {
    algorithm hmac-sha256;
    secret "base64-encoded-key";
};

启用DNSSEC：

dnssec-enable yes;
dnssec-validation auto;

常用管理命令

检查配置语法：

sudo named-checkconf

检查区域文件：

sudo named-checkzone example.com /etc/bind/db.example.com

重新加载配置：

sudo rndc reload

查看运行状态：

sudo rndc status

清空缓存：

sudo rndc flush

rndc

即 Remote Name Daemon Control

rndc 是 BIND9 DNS 服务器的远程管理工具，用于在不重启服务的情况下动态控制 named 守护进程（BIND9 的主进程）。 它通过加密的 TCP 连接（默认端口 953）与 named 通信，允许管理员执行各种管理操作。

核心功能

  • 重载配置 重新加载 named.conf 和更新的 zone 文件
  • 刷新缓存 清空 DNS 缓存
  • 动态更新 触发或冻结动态 DNS 更新
  • 状态查询 查看服务器运行状态
  • 日志控制 调整日志级别
  • 区域管理 手动同步或冻结 zone 数据
  • 安全控制 使用密钥认证，防止未授权访问

常用命令

基本操作

rndc reload         # 重新加载配置文件和 zone 数据
rndc reload <zone>  # 仅重载指定的 zone（如 rndc reload example.com）
rndc reconfig       # 仅重载 named.conf（不重新加载 zone 文件）
rndc flush          # 清空 DNS 缓存
rndc status         # 查看 BIND9 运行状态
rndc querylog       # 开启/关闭查询日志（on/off）

动态更新管理

rndc freeze <zone>        # 冻结 zone，禁止动态更新
rndc thaw <zone>          # 解冻 zone，允许动态更新
rndc sync -clean <zone>   # 强制同步动态更新到 zone 文件

调试 & 日志

rndc trace 3        # 提高日志级别（1-3，数字越大越详细）
rndc notrace        # 恢复默认日志级别
rndc dumpdb -cache  # 将缓存内容导出到 /var/cache/bind/named_dump.db

rndc 配置

密钥认证

rndc 默认使用 /etc/bind/rndc.key 进行身份验证：

key "rndc-key" {
    algorithm hmac-sha256;
    secret "Base64-encoded-key";
};

生成新密钥：

rndc-confgen -a -k <key-name> -c /etc/bind/rndc.key

named.conf 配置

确保 BIND9 允许 rndc 连接：

controls {
    inet 127.0.0.1 port 953
    allow { 127.0.0.1; } keys { "rndc-key"; };
};

常见问题

rndc: connection refused

原因：BIND9 未运行或 rndc 配置错误。

解决：

systemctl restart bind9
named-checkconf

rndc: permission denied

原因：密钥不匹配或权限问题。

解决：

chown bind:bind /etc/bind/rndc.key
chmod 640 /etc/bind/rndc.key

rndc reload 失败

原因：zone 文件语法错误。

解决：

named-checkzone example.com /etc/bind/db.example.com
