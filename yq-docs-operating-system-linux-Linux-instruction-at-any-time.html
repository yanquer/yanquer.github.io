ss

修改于: 2023-02-24

查看本地连接, 获取socket统计信息

ss命令可以传递的参数：

-a 显示所有套接字 -t 显示TCP端口 -u 显示UDP端口 -n 不解析主机名(即不转换服务端口号为服务名称) -r 显示路由套接字 -l 显示监听端口 -p 显示进程 -4 仅显示IPv4的socket连接 -x 显示更多信息,包括UNIX域套接字 -i 显示更多信息,包括接口信息 -o 显示更多信息,包括定时器信息

列含义

  • Recv-Q： 表示本端应用程序还没有从内核缓冲区（接收缓冲区）中读取的数据量（以字节为单位）。 - 简单说：数据已经通过网络到达你的机器，并被内核接收，但你的应用程序（例如 nginx, MySQL, 你的自定义服务）还没有调用 read() 或 recv() 系统调用将其取走。 - 理想情况： 这个值应该接近 0。如果这个值持续很大或不断增长，通常表明应用程序处理数据过慢，可能遇到了性能瓶颈（如 CPU 繁忙、程序卡死、或逻辑上有阻塞）。

  • Send-Q： 表示对端还没有确认（ACK）接收的数据量（以字节为单位） - 简单说：你的应用程序已经将数据交给内核发送，内核也已经发出，但还没有收到对端返回的 TCP 确认包（ACK）。 - 理想情况： 在网络通畅的情况下，这个值也应该很小。

    如果这个值持续很大或不断增长，通常表明网络链路或对端有问题。 比如网络延迟高、丢包严重，或者对端应用程序接收处理慢，导致其接收缓冲区满（进而通过 TCP 窗口通告机制通知本端停止发送）。

查看指定套接字:

ss -ln | grep 套接字名称
