Hook 的规则

只在最顶层使用Hook:

也就是不要在循环、条件或嵌套函数中调用Hook，这样可以做到各个hook 在每一次渲染中，调用的顺序是一致的.

那为什么要保证 hook 调用顺序? 和React实现hook的原理有关, 每次渲染时，React把所有调用的 hook 用数组来储存.

useEffect, useCallback, useMemo区别

参考
  • useEffect, useCallback, useMemo三者有何区别？
  • useMemo, useCallback, useEffect 三者区别
前置知识
组件重渲染的时机会比较 props 是否有更新
  • :useEffect : 在DOM更新完成后执行某些副作用操作，如数据获取，设置订阅以及手动更改 React 组件中的 DOM 等
  • :useMemo : 性能优化. useMemo返回的的是一个值，用于避免在每次渲染时都进行高开销的计算。
  • :useCallback : 性能优化, 返回一个函数，当把它返回的这个函数作为子组件使用时， 可以避免每次父组件更新时都重新渲染这个子组件

useCallback 和 useMemo 都是性能优化的手段， 类似于类组件中的 shouldComponentUpdate ，在子组件中使用 shouldComponentUpdate ， 判定该组件的 props 和 state 是否有变化，从而避免每次父组件 render 时都去重新渲染子组件。

setState界面未及时更新

参考: React 之 state 数据改变页面不更新未重新渲染的 7 种情况 及 解决办法

可能得原因

state 数据需要深更新

比如

// 错误方式
const obj = this.state.obj;
obj.forEach((e) => {
        xxxxx //省略的代码是改变了对象的值
});
this.setState({
    obj
});

解决办法: 浅拷贝方式获取对象并修改再赋值。

// 正确方式
const obj = [...this.state.obj];
obj.forEach((e) => {
        xxxxx //省略的代码是改变了对象的值
});
this.setState({
    obj
});

原因: 对象数组是引用方式 , 对于 react 来说它的值都是地址 (涉及到 tree diff)， 因为没有被重新赋值 (地址没有改变)，所以 react 会认为仍然是之前的元素 (element)，则不更新视图。

打印非实时

//默认string等于''
this.setState({
    string: '123'
})
console.log(this.state.string)//''

解决办法, 改用更新回调

this.setState({
    string: '123'
},()=>{
    //setState后的回调
    console.log(this.state.string)//'123'
})

原因: react 还没更新 state, 打印就执行完毕了。
