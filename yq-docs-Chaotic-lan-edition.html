语义化版本

修改于: 2024-04-02

比较流行的一个版本定义约束

语义化版本规则

版本格式:

主版本号.次版本号.修订号

版本号递增规则如下：

  • 主版本号 major：当你做了不兼容的 API 修改,
  • 次版本号 minor：当你做了向下兼容的功能性新增,
  • 修订号 patch：当你做了向下兼容的问题修正。

版本一般都是从 0.1.0 开始

语义化版本符号

比较时，可以使用以下比较符号:

=
  等于。用于指定必须匹配的确切版本号。
  例如：1.0.0 等于 1.0.0。

>
  大于。用于指定必须匹配的更高版本号。
  例如：1.0.0 大于 0.9.9。

>=
  大于或等于。用于指定必须匹配的更高或相同的版本号。
  例如：1.0.0 大于或等于 1.0.0。

<
  小于。用于指定必须匹配的更低版本号。
  例如：1.0.0 小于 1.1.0。

<=
  小于或等于。用于指定必须匹配的更低或相同的版本号。
  例如：1.0.0 小于或等于 1.0.0。

~
  约等于。用于指定与指定版本兼容的更新版本号，通常用于次版本号的更新。
  例如：~1.2.3 匹配 1.2.x 的任何版本，但不匹配 1.3.0。

^
  向上兼容。用于指定与指定版本兼容的更新版本号，但更严格，通常用于主版本号的更新。
  例如：^1.2.3 匹配 1.x.x 的任何版本，但不匹配 2.0.0。

*
  通配符。用于指定任何版本。
  例如：* 匹配任何版本号。

|| 或 ^
  用于范围选择，可以指定一个版本范围。
  例如：1.2.3 || 2.0.0 匹配 1.2.3 或 2.0.0。
  例如：^1.2.3 也可以用来表示 >=1.2.3 <2.0.0。

-
  指定一个版本号范围。
  例如：1.0.0 - 2.0.0

多个版本共同作用可以用空格分开, 如:

>1.1.0 <2.2.0
