一些坑

修改于: 2023-02-20

挂载本地卷不同步

使用 -v 挂载的卷不同步, 搜了一下大多都说是权限问题, 不认可...

重启容器未解决.

最后重启电脑解决了.

Mac下macvlan不可用

创建了一个network, (en0是mac无线网网卡)

docker network create --driver macvlan \
  --subnet=192.168.1.0/24 \
  --gateway=192.168.1.1 \
  -o parent=en0 \
  macvlan_network

compose 版本创建network

services:
  your_service:
    image: your_image
    container_name: your_container
    dns:  # 在这里设置DNS
      - 8.8.8.8
      - 8.8.4.4
    networks:
      macvlan_net:
        ipv4_address: 192.168.1.100

networks:
  macvlan_net:
    driver: macvlan
    driver_opts:
      parent: eth0
    ipam:
      config:
        - subnet: "192.168.1.0/24"
          gateway: "192.168.1.1"

会遇到报错: Error response from daemon: invalid subinterface vlan name en0, example formatting is eth0.10

搜了下, docker至今都不支持...

参考:
  • https://github.com/docker/for-mac/issues/3447
  • https://github.com/docker/for-mac/issues/3926
  • https://github.com/moby/libnetwork/issues/2614
  • 官网教程 https://docs.docker.com/engine/network/tutorials/macvlan/#other-networking-tutorials 提到了只支持 windows

Mac下host模式获取的ip错误

Docker on macOS 实际上是运行在 Linux VM（vmnetd） 里， host 模式共享的是 VM 的网络，而不是物理 Mac 的。

 # 获取容器是否是host
 docker inspect 容器名 | grep NetworkMode

 # 获取docker0的ip
 docker network inspect bridge | grep "IPv4Address"

 # 获取host的ip
 ifconfig en0 | grep inet

 # 获取host的ip
 ifconfig en0 | grep inet6

 # 获取host的ip
 ifconfig en0 | grep inet6 | awk '{print $3}'

查看容器 IP（应该和宿主机一致）

# 宿主机 IP
hostname -I

# 容器内 IP（应该和宿主机相同）
docker exec 容器名 hostname -I

在 macOS 上， host 模式的容器

  • IP = VM 的 IP`（如 `192.168.65.2 ），不是 Mac 本身的 IP（如 192.168.1.100 ）
