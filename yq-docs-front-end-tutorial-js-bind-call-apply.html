bind、call和apply 常用于函数试编程

bind

bind() 方法创建一个新的函数，当这个新函数被调用时，它的 this 值是传递给 bind() 的第一个参数，其余参数将作为新函数的参数供调用时使用。

语法:

fun.bind(thisArg[, arg1[, arg2[, ...]]])
参数
  • thisArg: 当绑定函数被调用时，该参数将作为原函数运行时的 this 指向。
  • arg1, arg2, ...: 当绑定函数被调用时，这些参数将作为原函数的参数传递。
thisArg说明
  • 如果 bind 函数的第一个参数是 null 或 undefined，那么 this 指向全局对象。
  • 在浏览器中，this 指向 window 对象。
  • 在 Node.js 中，this 指向 global 对象。
  • 同样适用于下面的 call 和 apply 方法

一个正常类调用的实例:

const Animal = {
  name: '猫',
  desc() {
    return `这是一只【${this.name}】`
  }
}

const Dog = {
  name: '狗'
}

console.log(Animal.desc()) // 这是一只【猫】

使用bind修改一个新的函数:

const dogDesc = Animal.desc.bind(Dog)
console.log(dogDesc()) // 这是一只【狗】

call

call() 方法调用一个函数，其具有一个指定的 this 值和分别地提供的参数（参数的列表）。

语法:

fun.call(thisArg[, arg1[, arg2[, ...]]])

参数说明与bind一致

与bind的区别
  • call() 方法不会创建一个新的函数，而是立即执行。

还是上面的Animal例子:

console.log(Animal.desc.call(Dog)) // 这是一只【狗】

apply

apply() 方法调用一个函数，其具有一个指定的 this 值，以及作为一个数组（或类数组对象）提供的参数。

语法:

fun.apply(thisArg, [argsArray])
参数说明
  • thisArg: 当调用 fun 函数时，this 的值将被设置为 thisArg。
  • argsArray: 一个数组或类数组对象，其中的数组元素将作为单独的参数传给 fun 函数。
与bind的区别
  • apply() 方法不会创建一个新的函数，而是立即执行。
与call的区别
  • call() 方法的参数是一个列表，而 apply() 方法的参数是一个数组。

还是上面的Animal例子:

console.log(Animal.desc.apply(Dog)) // 这是一只【狗】

拓展-TS装饰器中的应用

例子:

// target: 被装饰的类实例
// key: 被装饰的方法名
// descriptor: 被装饰的方法的描述对象, 包含value、writable、enumerable、configurable等属性
//  - value: 被装饰的方法指针引用
//  - writable: 是否可写
//  - enumerable: 是否可枚举
//  - configurable: 是否可配置
function log(target: any, key: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value
  descriptor.value = function (...args: any[]) {
    console.log(`Arguments: ${args}`)
    return originalMethod.apply(this, args)
  }
  return descriptor
}

class Dog {
  @log
  bark() {
    console.log('汪汪')
  }
}

const dog = new Dog()
dog.bark()

注解

再拓展一下箭头函数, 箭头函数没有自己的this, 会继承外层的this, 所以在箭头函数中使用bind、call、apply是无效的
