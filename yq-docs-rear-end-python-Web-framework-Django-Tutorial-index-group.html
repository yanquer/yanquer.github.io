数据库分组查询

修改于: 2025-10-30

Django ORM 支持分组查询, 需要注意没有显示的 group by 关键字, 而是通过 values() 和 annotate() 来实现.

比如:

from django.db.models import Count

Model.objects.values('field1').annotate(count=Count('field1'))

上面代码会根据 field1 字段进行分组, 并统计每个分组的数量, 返回结果是一个包含字典的 QuerySet, 每个字典包含 field1 和 count 两个键值对.

等价的SQL 语句:

SELECT field1, COUNT(field1) as count
FROM app_model
GROUP BY field1;

如果需要对分组结果进行过滤, 可以使用 filter() 方法, 例如过滤出数量大于 5 的分组:

Model.objects.values('field1').annotate(count=Count('field1')).filter(count__gt=5)

如果分组时需要返回原始模型对象, 可以省略 values() 方法, 例如:

Model.objects.annotate(count=Count('pk')).filter(count__gt=5).order_by('field1')

此处并没有使用 values() 方法, 所以返回的结果是包含模型对象的 QuerySet, 每个对象都有一个额外的 count 属性, 末尾使用 order_by() 方法对结果进行排序, 方便在python的角度整合分组结果.

等价的SQL 语句:

SELECT *, COUNT(pk) as count
FROM app_model
GROUP BY field1
HAVING COUNT(pk) > 5
ORDER BY field1;

注意这里其实没有分组了, 是排序后在python端进行手动分组的效果.

补充: aggregate 与 annotate 的作用与区别

aggregate()

用于对 整个 QuerySet 进行聚合计算, 返回一个包含聚合结果的字典. 例如计算总数、平均值、最大值等.

例子:

from django.db.models import Avg

result = Model.objects.aggregate(avg_field1=Avg('field1'))
# result 是一个字典, 包含键 'avg_field1'
annotate()

用于对 QuerySet 中的每个对象 进行聚合计算, 返回一个包含原始对象和聚合结果的 QuerySet. 例如计算每个分组的总数、平均值等.

例子:

from django.db.models import Count

queryset = Model.objects.values('field1').annotate(count=Count('field1'))
# queryset 是一个包含字典的 QuerySet, 每个字典包含 'field1' 和 'count'

某些情况下, 两个其实都可以实现一样的效果, 区别是annotate需要先做一个中间字段
