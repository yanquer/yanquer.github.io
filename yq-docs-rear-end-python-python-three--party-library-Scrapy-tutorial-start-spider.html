参考: Python爬虫：Scrapy从脚本运行爬虫的5种方式

在 使用 的例子中, 是直接在命令行运行的, 方便是方便, 缺点是无法 Debug.

大体上, 运行爬虫有以下方式, 主要分成两大类

  • 命令行启动
  • py代码启动

命令行启动

先去项目根目录

# 若需要导出, scrapy crawl itcast -O t.json
$ scrapy crawl itcast

若在没有创建项目的情况下运行爬虫

$ scrapy runspider itcast.py

py代码启动

CrawlerProcess(建议)

多个执行是并行的

from scrapy import Spider
from scrapy.crawler import CrawlerProcess
from scrapy.utils.project import get_project_settings

class BaiduSpider(Spider):
    ...

if __name__ == '__main__':
    # 通过方法 get_project_settings() 获取配置信息
    process = CrawlerProcess(get_project_settings())
    process.crawl(BaiduSpider)
    # 如果有多个, 继续 crawl
    #   如 process.crawl(SinaSpider)
    #   是基本并行启动的
    process.start()

CrawlerRunner(官方建议)

多个执行是顺序的

from scrapy import Spider
from scrapy.crawler import CrawlerRunner
from scrapy.utils.log import configure_logging
from twisted.internet import reactor


class BaiduSpider(Spider):
    ...


if __name__ == '__main__':
    # 直接运行控制台没有日志
    configure_logging(
        {
            'LOG_FORMAT': '%(message)s'
        }
    )

    runner = CrawlerRunner()

    d = runner.crawl(BaiduSpider)
    # 如果有多个
    #   runner.crawl(SinaSpider)
    #   d = runner.join()
    #   这样是顺序启动的

    d.addBoth(lambda _: reactor.stop())

    reactor.run()

cmdline

算是一种另类的命令行启动

缺点是无法在单线程中并行多个爬虫, 因为如果有多行 cmdline.execute, 第一个执行完毕就直接退出了.

使用 cmdline 本质其实还是命令行执行的

from scrapy import cmdline, Spider


class BaiduSpider(Spider):
    name = 'baidu'

    start_urls = ['http://baidu.com/']

    def parse(self, response):
        self.log("run baidu")


if __name__ == '__main__':
    cmdline.execute("scrapy crawl baidu".split())
