Scrapy 预定义的 pipelines

  • ImagesPipeline

ImagesPipeline

ImagesPipeline 是 Scrapy 内置的一个 pipeline，用于下载图片。

注解

另外需要注意的是, 有时会发现即使在 ITEM_PIPELINES 中加入了 scrapy.pipelines.images.ImagesPipeline, 实际没有生效也没有报错, 可能是因为没有安装 Pillow 库, 需要手动安装:

pip install Pillow

简单使用

使用方式, 简单使用就是在 settings.py 中配置:

ITEM_PIPELINES = {
    'scrapy.pipelines.images.ImagesPipeline': 1
}

# 存储路径
IMAGES_STORE = '/path/to/your/images'

然后在 items.py 中定义一个字段:

import scrapy

class MyItem(scrapy.Item):
    image_urls = scrapy.Field()
    images = scrapy.Field()

可以通过 IMAGES_URLS_FIELD 和 IMAGES_RESULT_FIELD 来自定义。 比如:

IMAGES_URLS_FIELD = 'image_urls'
IMAGES_RESULT_FIELD = 'images'

其中, image_urls 用于存放图片的 url，images 用于存放下载后的图片信息。 默认情况下路径是由 SHA1 hash 算法生成

自定义使用

继承 ImagesPipeline，重写方法:

from scrapy.pipelines.images import ImagesPipeline

class MyImagesPipeline(ImagesPipeline):

    # 自定义请求方式
    def get_media_requests(self, item, info):
        return [scrapy.Request(x, meta={'image_name': item.get('image_name')}) for x in item.get(self.images_urls_field, [])]

    # 自定义文件路径
    def file_path(self, request, response=None, info=None):
        return 'full/%s.jpg' % request.meta['image_name']

    # 自定义下载完成后的处理
    def item_completed(self, results, item, info):
        image_paths = [x['path'] for ok, x in results if ok]
        if not image_paths:
            raise DropItem("Item contains no images")
        item['image_paths'] = image_paths
        return item

然后在 settings.py 中配置:

ITEM_PIPELINES = {
    'myproject.pipelines.MyImagesPipeline': 1
}

# 存储路径
IMAGES_STORE = '/path/to/your/images'

一个完整的MyImagesPipeline例子:
