parcel

修改于: 2025-02-05
相关资源
  • npm地址: https://www.npmjs.com/package/parcel
  • 文档: https://parceljs.org/docs/

快速, 零配置的web应用打包工具

安装:

npm install --save-dev parcel
# yarn add --dev parcel

使用

Web App

比如入口是 src/index.html

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8"/>
    <title>My First Parcel App</title>
    <link rel="stylesheet" href="styles.css" />
    <script type="module" src="app.js"></script>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

编译:

yarn parcel src/index.html

# Or when using npm run:
# npx parcel src/index.html

package.json

{
  "name": "my-project",
  "source": "src/index.html",
  "scripts": {
    "start": "parcel",
    "build": "parcel build"
  },
  "devDependencies": {
    "parcel": "latest"
  }
}

library

比如入口是 src/index.js

export function add(a, b) {
  return a + b;
}

package.json

{
  "name": "my-library",
  "version": "1.0.0",
  "source": "src/index.js",
  "main": "dist/main.js",
  "scripts": {
    "watch": "parcel watch",
    "build": "parcel build"
  },
  "devDependencies": {
    "parcel": "latest"
  }
}

注解

默认只需要指定 source 和 main 即可, more的main表示CommonJS模块的入口

如果还需要生成其它格式的输出, 可以按照以下修改/增加:

// ES module
"module": "dist/module.js",
// CommonJS
"cjs": "dist/cjs.js",
// UMD
"umd": "dist/umd.js",
// SystemJS
"system": "dist/system.js",
// AMD
"amd": "dist/amd.js",
// ESM
"esm": "dist/esm.js",
// IIFE
"iife": "dist/iife.js",
// ESM with browser field
"browser": "dist/browser.js",

问题

build时候报错内存错误

参考
  • https://github.com/parcel-bundler/parcel/issues/9989
  • https://github.com/parcel-bundler/parcel/issues/9976
  • https://github.com/parcel-bundler/parcel/issues/9926

可能的输出:

node(73192,0x172977000) malloc: Incorrect checksum for freed object 0x1228ef800: probably modified after being freed.
Corrupt value: 0x0
node(73192,0x172977000) malloc: *** set a breakpoint in malloc_error_break to debug
Abort trap: 6

大部分出现与使用 node-v22 版本, 是node的问题, 升级到 22.10 及以上版本即可, 或者降级也行.

其它

缺点是没找打怎么内联三方库打包, 无奈转到 rollup
