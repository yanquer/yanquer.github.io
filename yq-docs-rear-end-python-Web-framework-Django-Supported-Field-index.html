Django支持的Field

修改于: 2024-03-09

Field公共关键字参数

on_delete

当关联对象被删除时, 应该如何处理当前对象

models.CASCADE (级联删除)
  • 当被引用的对象删除时，也删除所有包含外键指向它的对象
  • 这是最常用的选项
models.PROTECT (保护)
  • 防止删除被引用的对象，如果还有对象引用它，会引发 ProtectedError
  • 用于防止意外删除重要数据
models.SET_NULL (设为 NULL)
  • 当被引用的对象删除时，将外键设置为 NULL
  • 只有在外键字段设置了 null=True 时才可用
models.SET_DEFAULT (设为默认值)
  • 当被引用的对象删除时，将外键设置为默认值
  • 必须为外键字段定义 default 参数
models.SET() (设为指定值)
  • 当被引用的对象删除时，将外键设置为传递给 SET() 的值
  • 可以是一个值或可调用对象

例如: on_delete=models.SET(get_default_user)

models.DO_NOTHING (不采取任何动作)
  • 数据库不执行任何操作
  • 如果数据库强制引用完整性，这会导致错误
  • 需要自己在数据库层面处理

on_delete示例

当 Author 表中的某个作者被删除时， Book 表中所有引用该作者的记录的 author 字段将被设置为 NULL 。

from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(
        Author,
        on_delete=models.SET_NULL,  # 当作者被删除时，将 author 字段设为 NULL
        null=True,
        blank=True
    )
