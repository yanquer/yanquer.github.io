常见宏

修改于: 2025-03-17

有时候会看到这个的定义

#[derive(Default, Debug, PartialEq)]
struct Config {
    value: i32,
}

研究了一下 derive 这个宏, 以下是一些常见的宏

宏

宏是 Rust 中的一种特殊类型的函数，用于生成代码。宏允许你定义一个模板，然后使用这个模板来生成代码。

宏的语法如下：

macro_rules! macro_name {
    (pattern) => (expansion);
    (pattern) => (expansion);
    ...
}

其中，pattern 是宏的匹配模式，expansion 是宏的扩展代码。

部分常见的宏

  • derive

derive

比如 #[derive(Default, Debug, PartialEq)]` 是 Rust 中的一个属性宏（attribute macro），用于为结构体或枚举自动实现某些 trait。 具体来说，它可以帮助你自动生成代码，而无需手动实现这些 trait。

以下是 #[derive(Default, Debug, PartialEq)] 中每个 trait 的作用：

注解

  • Default: 提供默认值。
  • Debug: 支持调试输出（{:?}）。
  • PartialEq: 支持相等比较（== 和 !=）。

通过 #[derive(...)]，Rust 可以自动生成这些 trait 的实现，减少重复代码，提高开发效率。 如果结构体的字段也实现了这些 trait，那么 #[derive(...)] 会自动递归地为结构体生成实现。

Default

Default trait 为类型提供了一个默认值。

通过 #[derive(Default)] ，Rust 会自动为结构体生成一个 Default 实现，其中每个字段都会使用其自身的 Default 实现来初始化。

示例

#[derive(Default)]
struct Config {
    value: i32,
    name: String,
}

fn main() {
    let config = Config::default();
    println!("value: {}, name: {}", config.value, config.name);
}

输出:

value: 0, name:
  • value 是 i32 类型，它的默认值是 0。
  • name 是 String 类型，它的默认值是空字符串 ""。

如果没有 #[derive(Default)]，你需要手动实现 Default trait：

impl Default for Config {
    fn default() -> Self {
        Config {
            value: 0,
            name: String::new(),
        }
    }
}

Debug

Debug trait 用于格式化输出，通常用于调试。

通过 #[derive(Debug)] ，Rust 会自动为结构体生成一个 Debug 实现，使得你可以使用 {:?} 或 {:#?} 来打印结构体的内容。

示例：

#[derive(Debug)]
struct Config {
    value: i32,
    name: String,
}

fn main() {
    let config = Config {
        value: 42,
        name: String::from("example"),
    };
    println!("{:?}", config);
}

输出:

Config { value: 42, name: "example" }

如果没有 #[derive(Debug)] ， 你需要手动实现 Debug trait，或者无法使用 {:?} 打印结构体。

PartialEq

PartialEq trait 用于比较两个值是否相等。

通过 #[derive(PartialEq)] ，Rust 会自动为结构体生成一个 PartialEq 实现，使得你可以使用 == 和 != 来比较两个结构体的实例。

示例：

#[derive(PartialEq)]
struct Config {
    value: i32,
    name: String,
}

fn main() {
    let config1 = Config {
        value: 42,
        name: String::from("example"),
    };
    let config2 = Config {
        value: 42,
        name: String::from("example"),
    };
    println!("Are they equal? {}", config1 == config2);
}

输出:

Are they equal? true

如果没有 #[derive(PartialEq)] ，你需要手动实现 PartialEq trait，或者无法使用 == 和 != 进行比较。

组合使用 #[derive(Default, Debug, PartialEq)]

通常我们会将多个 trait 组合在一起使用，以便为结构体提供更多的功能。例如：

#[derive(Default, Debug, PartialEq)]
struct Config {
    value: i32,
    name: String,
}

fn main() {
    let config1 = Config::default();
    let config2 = Config {
        value: 0,
        name: String::new(),
    };

    println!("{:?}", config1); // 使用 Debug 打印
    println!("Are they equal? {}", config1 == config2); // 使用 PartialEq 比较
}

输出:

Config { value: 0, name: "" }
Are they equal? true
