adrf 基于 djangorestframework , 额外提供了异步支持(因为 drf 是同步的且官方几年都没打算要支持异步)

安装:

pip install adrf

然后在 settings.py 中增加/修改设置如下:

INSTALLED_APPS = [
    ...
    'adrf',
]

相对于同步的 drf, 需要将继承的序列化类更新为 from adrf import serializers as adrf_serializers

from adrf import serializers as adrf_serializers
from rest_framework import serializers

class AuthorSerializer(adrf_serializers.ModelSerializer):

  class Meta:
    model = Author
    fields = '__all__'

注意当存在嵌套序列化时, 多对多字段的嵌套与 drf 一致, 但是外键字段需要重写, 因为外键的直接 .get 不能在异步上下文中使用,

一个例子

class Author(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()

class BookInfo(models.Model):
    title = models.CharField(max_length=100)
    author = models.ForeignKey(Author, on_delete=models.CASCADE)
    publish_date = models.DateField()

from adrf import serializers as adrf_serializers
from adrf import fields as adrf_fields
from rest_framework import serializers

class AuthorSerializer(adrf_serializers.ModelSerializer):

  class Meta:
    model = Author
    fields = '__all__'

class BookInfoSerializer(adrf_serializers.ModelSerializer):

  # 异步上下文会报错
  # author = AuthorSerializer(read_only=True)
  # 需要重写
  author = adrf_fields.SerializerMethodField()

  class Meta:
    model = BookInfo
    fields = '__all__'

  async def get_author(self, obj: BookInfo):
      # _author = await Author.objects.aget(id=obj.author_id)
      # 使用 filter 可以避免外键为空时候的报错
      _author = await Author.objects.filter(id=obj.author_id).afirst()
      if _author:
          return await AuthorSerializer(instance=_author).ato_representation(_author)
      return None

若存在某些旧的同步代码不方便全部修改为异步, 可以使用 django 的 asgiref.sync.sync_to_async 装饰器来将同步代码转换为异步

from asgiref.sync import sync_to_async

@sync_to_async
def do_something():
  ...

# 在异步函数中调用
await do_something()
