前端的测试框架, 支持ui测试, 单元测试等.

安装:

npm i jest

环境配置

在项目根目录下创建一个 jest.config.js 文件, 用于配置jest的环境.

或者直接在 package.json 中添加配置项:

"jest": {
  "testEnvironment": "jsdom"
}

testEnvironment 可以设置为 node 或 jsdom. 默认为 jsdom.

  • node 使用node环境
  • jsdom 使用浏览器环境

配置(可选)

  • testEnvironment: 测试环境, 默认为 jsdom
  • testMatch: 测试文件匹配规则, 默认为 ["**/__tests__/**/*.[jt]s?(x)", "**/?(*.)+(spec|test).[tj]s?(x)"]
  • testPathIgnorePatterns: 忽略测试文件匹配规则, 默认为 ["/node_modules/"]
  • transform: 转换器, 默认为 { "^.+\.jsx?$": "babel-jest" }
  • transformIgnorePatterns: 忽略转换器匹配规则, 默认为 ["/node_modules/"]
  • moduleFileExtensions: 模块文件扩展名, 默认为 ["js", "jsx", "json", "node"]
  • moduleNameMapper: 模块名称映射, 默认为 {}
  • setupFilesAfterEnv: 测试环境初始化文件, 默认为 []
  • setupFiles: 测试环境初始化文件, 默认为 []
  • globalSetup: 全局初始化文件, 默认为 []
  • globalTeardown: 全局销毁文件, 默认为 []
  • coverageDirectory: 覆盖率报告目录, 默认为 coverage
  • coveragePathIgnorePatterns: 覆盖率忽略文件匹配规则, 默认为 ["/node_modules/"]
  • coverageReporters: 覆盖率报告格式, 默认为 ["json", "text", "lcov", "clover"]
  • coverageThreshold: 覆盖率阈值, 默认为 {}
  • collectCoverage: 是否收集覆盖率, 默认为 false
  • collectCoverageFrom: 收集覆盖率文件匹配规则, 默认为 ["**/*.{js,jsx,ts,tsx}", "!**/node_modules/**"]
  • collectCoverageOnlyFrom: 收集覆盖率文件匹配规则, 默认为 []
  • coverageProvider: 覆盖率提供者, 默认为 babel

配置文件

配置文件支持js格式/ts格式/json文件, 也可以直接放在 package.json 中.

js文件

ts文件

可以在 jest.config.ts 文件中添加以下配置(或者直接在 package.json 中添加):

import type {Config} from 'jest';
import {defaults} from 'jest-config';

export default async (): Promise<Config> => {
  return {
    moduleFileExtensions: [...defaults.moduleFileExtensions, 'mts'],
    verbose: true,
  };
};

注解

也支持 jest.config.json 文件

{
  "bail": 1,
  "verbose": true
}

或者，可以在项目 package.json 里的 "jest" 键进行配置 Jest，如：

{
  "name": "my-project",
  "jest": {
    "verbose": true
  }
}

ts支持

有两种方式

  • 使用 ts-jest 进行编译, 参考: ts-jest
  • 使用 babel 进行编译

基于babel的配置

安装依赖

yarn add --dev @babel/preset-typescript

在 babel.config.js 中添加以下配置:

module.exports = {
  presets: [
    ['@babel/preset-env', {targets: {node: 'current'}}],
    '@babel/preset-typescript',
  ],
};

注解

由于Babel中的TypScript支持纯粹是转译，因此Jest不会在您的测试运行时对其进行类型检查。 如果您想要这样做，您可以使用 ts-jest ，或者只是单独运行TypScript编译器tsc（或作为构建过程的一部分）。

可能遇到的报错

jest Cannot use import statement outside a module

情况1: 没有使用ESM外部模块

参考: Jest - SyntaxError: 无法在模块外使用 import 语句

没有使用ESM外部模块时, 多半就是是否配置 transform

检查jest.config.ts中配置项是否指定了ts文件的转译行为:

{
  "transform": {
    "^.+\\.tsx?$": "ts-jest"
  }
}

附 tsconfig.json

{
  "compilerOptions": {
    "target": "es2016",
    "module": "commonjs",

    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "strict": true,
    "skipLibCheck": true,
    "outDir": "dist"
  },
  "include": ["src"]
}

情况2: 使用了ESM外部模块

比如, 使用了 chalk 模块会遇到:

node_modules/chalk/source/index.js:1
  ({"Object.<anonymous>":function(module,exports,require,__dirname,__filename,jest){import ansiStyles from '#ansi-styles';
                                                                                    ^^^^^^

  SyntaxError: Cannot use import statement outside a module
import chalk from 'chalk'

chalk5.x 已经全系列支持ESM了, 但是 jest 还不支持ESM模块, 可以选择降级到 4.x 的 commonjs 版本

如果不想降级, 需要配置对 ESM 的支持, tsconfig.json 需要添加 module: "ESNext" 和 target: "ES2020" 选项 (ES系都行),

{
  "compilerOptions": {

    // 关键配置
    "target": "ES2020",
    "module": "ESNext",
    // node 项是为了编辑器能正常解析 node_modules 中的模块
    "moduleResolution": "node",
    // 关键配置结束

    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "strict": true,
    "skipLibCheck": true,
    "outDir": "dist"
  },
  "include": ["src"]
}

jest.config.ts 需要配置

import type {Config} from 'jest';

export default async (): Promise<Config> => {
    return {

        // 关键配置：处理 ESM 模块（如 chalk 5+）
        extensionsToTreatAsEsm: ['.ts'],
        // moduleNameMapper: {
        //     '^(\\.{1,2}/.*)\\.js$': '$1',
        //     '#ansi-styles': '<rootDir>/node_modules/chalk/source/vendor/ansi-styles/index.js'
        // },

        transform: {
            "^.+\.tsx?$": ["ts-jest",{
                useESM: true,  // 启用 ESM 支持
                isolatedModules: true  // 避免 TS 类型检查影响测试速度
            }],
        },
        // 排除 node_modules，但允许转换 chalk
        // transformIgnorePatterns: [
        //     'node_modules/(?!chalk/)'
        // ]

    } as Config;
};

另外, node启动时需要添加 --experimental-vm-modules 选项:

"scripts": {
  "test": "NODE_OPTIONS=--experimental-vm-modules jest"
},
