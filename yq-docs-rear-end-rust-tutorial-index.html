str 与 String

当函数参数列表是 str 时, 其实也可以传递 String 的参数

主要区别
内存分配:
  • String 是一个动态分配的 Unicode 字符串类型, 它拥有自己的内存并可以被修改。
  • str 是一个静态分配的字符串切片类型, 它不拥有内存, 而是指向其他字符串的一部分。
所有权:
  • String 是一个拥有所有权的类型,它可以被移动和克隆。
  • str 是一个借用类型,它不拥有任何数据的所有权。
长度:
  • String 的长度可以在运行时动态改变。
  • str 的长度在编译时就确定,无法动态改变。
使用场景:
  • String 通常用于需要动态修改或者接受用户输入的字符串。
  • str 通常用于函数参数、返回值或者其他静态已知长度的字符串。

总的来说, String 提供了更多的灵活性和功能,但需要消耗更多的内存和资源。 而 str 则更加轻量和高效,但只能用于不可变的字符串。在编写 Rust 代码时,需要根据具体的需求选择合适的字符串类型。
