npm包增加私有git库

修改于: 2024-07-30

有时, 需要使用私有的git仓库作为npm包, 但是由于私有仓库的原因, 无法直接使用

增加github的开源库

有时候发现用的npm包存在问题, 但是由于维护者不更新等原因, 不得不自行fork一个仓库用.

此处贴一下使用私有git仓库作为npm包的历程

  1. 先fork旧的仓库

若需要配置仅使用某个文件夹, 在此fork仓库的 package.json 配置

"files": [
  "dist"
],

若 dist 需要先编译才能生成, 在 package.json 的 scripts 配置一个编译处理

"scripts": {
  "build": "echo build && rollup -c --bundleConfigAsCjs",
  "prepare": "npm run build"
}

prepare 会在 npm install 时执行

关于 package.json 配置参考 :package.json配置说明

  1. 在自己项目package.json中配置需要用的仓库

比如需要用 https://github.com/xxxx/repo.git 的 v1.0.0 版本

"dependencies": {
  "repo": "https://github.com/xxxx/repo.git#v1.0.0"
}

增加自己github的私有仓库-方案一

直接用ssh添加自己的私有库, 比如

yarn add git+ssh://git@github.com/yanquer/YQFrontCommon.git#v0.2.0
缺点
  • 会拉整个仓库, package.json 中的 files 配置无效
  • 需要配置ssh

增加自己github的私有仓库-方案二

参考
  • [简书] 如何使用GitHub Packages创建npm私有库
  • [CSDN] 在发布 npm 包的时候提示没有登录，解决办法 This command requires you to be logged in to https://npm.pkg.github.com
  • [GitHub] npm publish has resulted in error "You need to authorize this machine using npm adduser"

发布到github的npm仓库: https://npm.pkg.github.com/

github权限配置

github上配置私有token

新建时的选项

生成后的token注意保存, 只能显示一次, 不可再次查看

然后在本地登陆这个npm

$ npm login --registry=https://npm.pkg.github.com
> Username: USERNAME
> Password: TOKEN
> Email: PUBLIC-EMAIL-ADDRESS

注解

只能用 npm, 不能用 yarn, 暂时不支持

npm包创建

(建立包就不用说了)

在自己的包仓库下加一个 .npmrc 文件, 内容如下:

registry=https://npm.pkg.github.com/yanquer/

如果上面没有手动登陆的话, 需要在 .npmrc 中添加token:

registry=https://npm.pkg.github.com/yanquer/
always-auth=true
//npm.pkg.github.com/:_authToken=TOKEN

注解

后续包建好后, 其它仓库如果要使用这个包, 也要加一个 .npmrc 文件:

registry=https://npm.pkg.github.com/yanquer/

更新package.json

npm init --scope=yanquer

如果生成的 package.json 中没有 publishConfig 字段, 需要手动添加

"publishConfig": {
  "registry": "https://npm.pkg.github.com"
}

否则会报错 This command requires you to be logged in to https://npm.pkg.github.com, 即使上面登陆步骤执行了)

然后发布即可

npm publish

npm notice
npm notice 📦  @yanquer/yq-front-common@0.2.0
npm notice Tarball Contents
npm notice 56B README.md
npm notice 1.9kB package.json
...
npm notice total files: 56
npm notice
npm notice Publishing to https://npm.pkg.github.com with tag latest and default access

增加自己github的私有仓库-方案二-多包模式

即在一个仓库中包含多个包, 然后发布到github的npm仓库中

说明

  • 只需要在根目录下添加一个 .npmrc 文件,

    内容如下:

    registry=https://npm.pkg.github.com/

注意一定是 registry 不要写成安装时候用的 @yanquer:registry 的格式, 否则会识别用户信息错误

根目录的package.json中设置工作空间

"workspaces": [
  "packages/*"
]

每个包中设置单独的package.json记得设置发布地址

"publishConfig": {
  "registry": "https://npm.pkg.github.com/"
}

发布可以单独进入每一个包然后执行

npm publish

或者批量发布

npm publish --workspaces
