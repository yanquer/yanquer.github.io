这个其实可以分成两个部分, CORS 和 CSRF

  • CORS 是跨域资源共享, 用于解决跨域问题
  • CSRF 是跨站请求伪造, 用于解决跨站请求伪造问题

跨域访问配置

先说下如何设置跨域访问, 先安装 django-cors-headers

pip install django-cors-headers

然后在 settings.py 中增加/修改设置如下

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',

    # 新增的应用
    'corsheaders',
]

MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',

    # 新增的中间件 放在 CommonMiddleware 之前
    'corsheaders.middleware.CorsMiddleware',

    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]

# 允许所有的跨域请求
CORS_ORIGIN_ALLOW_ALL = True
# 是否允许ajax跨域请求时携带cookie
CORS_ALLOW_CREDENTIALS = True

# 也可以使用白名单
# 旧版本是 CORS_ORIGIN_WHITELIST, 新版本是 CORS_ALLOWED_ORIGINS
CORS_ALLOWED_ORIGINS = [
    'http://127.0.0.1:5173',
    'http://localhost:5173',
]

跨站请求伪造配置

Django 默认开启了 CSRF 防护, 目前官方最新文档参考 如何使用 Django 提供的 CSRF 防护功能-CSRF , 但是在前后端分离的情况下, 前端无法直接拿到 csrf_token, 所以需要特殊处理.

网上搜了一下, 就没有一个完善的不关闭 CSRF 防护的方案, 官方文档也没说明白.

注解

这里得提一下, 如果仅设置 CORS 不配置 CSRF 会出现什么情况

  • 普通请求比如正常的 GET OPTIONS 请求没问题
  • 复杂的请求比如 POST 请求会出现 403 错误, 可能会看到 CSRF cookie not set 或 CSRF token missing or incorrect

网上主流方式1-关闭防护(不推荐)

在 settings.py 中关闭 CSRF 防护, 注释掉这个中间件:

MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',

    # 注释掉这里
    # 'django.middleware.csrf.CsrfViewMiddleware',

    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]

网上主流方式2-关闭指定视图/接口的防护(不推荐)

在视图/接口中关闭 CSRF 防护:

from django.views.decorators.csrf import csrf_exempt

@csrf_exempt
def my_view(request):
    return HttpResponse('Hello, World!')

自定义获取csrf_token(推荐)

此方案基于官方文档 当 CSRF_USE_SESSIONS 或 CSRF_COOKIE_HTTPONLY 为 True 时获取令牌

后端配置

首先是在 settings.py 配置:

# 将信任的域名加入到 CSRF_TRUSTED_ORIGINS 中, 此处与上面的 CORS_ALLOWED_ORIGINS 一致
CSRF_TRUSTED_ORIGINS = CORS_ALLOWED_ORIGINS
# 允许跨域请求时携带cookie的域名, 此处与上面的 CORS_ALLOWED_ORIGINS 一致
# CSRF_COOKIE_DOMAIN = CORS_ALLOWED_ORIGINS

# 服务器创建的 csrf_token 的属性
# 是否将 CSRF 标记存储在用户的会话中，而不是 cookie 中。这需要使用 django.contrib.session。
# CSRF_USE_SESSIONS = True    # 此项不设置
# 是否对 CSRF cookie 使用 HttpOnly 标志。如果设置为 True，客户端的 JavaScript 将无法访问 CSRF cookie
# CSRF_COOKIE_HTTPONLY = True
# CSRF cookie 上 SameSite 标志的值。该标志可防止在跨站点请求中发送 cookie。 默认 'Lax'
# CSRF_COOKIE_SAMESITE = "None"
# 是否为 CSRF cookie 使用安全 cookie。如果设置为 True，cookie 将被标记为 安全，这意味着浏览器可以确保 cookie 只在 HTTPS 连接下发送。
# CSRF_COOKIE_SECURE = True

简而言之其实只需要:

CSRF_TRUSTED_ORIGINS = CORS_ALLOWED_ORIGINS

然后是定义一个接口可以让前端获取到 csrf_token:

from django.middleware.csrf import get_token
from django.http import JsonResponse

def get_csrf_token(request):
    """ 获取 csrftoken, 跨域这样获取貌似有点毛病... 后续需要加强安全性"""

    csrf_token = get_token(request)
    return JsonResponse({'csrf_token': csrf_token})
    # return HttpResponse(json.dumps({'token': token}), content_type="application/json,charset=utf-8")

定义接口url:

path('get_csrf_token/', views.get_csrf_token, name='get_csrf_token'),

前端配置

前端获取 csrf_token 的接口:

import axios from 'axios'

// 获取 csrf_token
export const getCsrfToken = async (): string => {
  return (await axios.get('xxx/get_csrf_token/')).data.csrf_token
}

然后当请求到达的时候, 手动设置 cookie:

import axios from 'axios'
import Cookies from 'js-cookie'

// 获取 csrf_token
export const getCsrfToken = async (): string => {
  return (await axios.get('get_csrf_token/')).data.csrf_token
}

// 设置 csrf_token
export const setCsrfToken = async (data): void => {
  const csrfToken = await getCsrfToken()
  // 必须要设置 cookie
  Cookies.set("csrftoken", csrfToken ?? "", {
      sameSite: 'Lax', // 或 'Strict', 'None' 根据需求
      // secure: false,     // 如果使用 HTTPS，设置为 true. (貌似需要与服务器保持一致)
      // expires: 7,       // 可选：设置过期时间
      path: "/",
  });
  // 访问时候配置 config 的 headers, 必须有 `X-CSRFToken`
  // axios.defaults.headers.common['X-CSRFToken'] = csrfToken
  // 比如访问 post 的 api/add 接口
  //  withCredentials = true; 表示允许携带cookie
  return await axios.post('api/add', data, { headers: { 'X-CSRFToken': csrfToken, withCredentials: true } })
}
