打包生成目标区别

修改于: 2025-02-10

前端打包生成的目标有

  • CommonJS / CJS
  • ES Module / ESM
  • UMD
  • AMD
  • CMD
  • SystemJS
  • IIFE

CommonJS / CJS

CommonJS 是 Node.js 默认的模块系统，用于在 Node.js 环境中加载模块。CommonJS 模块是同步加载的，只有加载完成后才能执行后续代码。

有四个重要的环境变量为模块化的实现提供支持:

module、exports、require、global

ES Module / ESM

ES Module 是 ECMAScript 6 引入的模块系统，用于在浏览器环境中加载模块。ES Module 模块是异步加载的，可以并行加载多个模块。

AMD

AMD (Asynchronous Module Definition) 是一种异步模块定义规范，用于在浏览器环境中加载模块。AMD 模块是异步加载的，可以并行加载多个模块。

一个常见用法是使用 RequireJS 加载 AMD 模块:

  • 用require.config()指定引用路径等，
  • 用definde()定义模块，
  • 用require()加载模块。

CMD

CMD (Common Module Definition) 是一种通用模块定义规范，用于在浏览器环境中加载模块。CMD 模块是同步加载的，只有加载完成后才能执行后续代码。

SystemJS

SystemJS 是一个模块加载器，支持加载 CommonJS、AMD、ES Module 等多种模块格式。SystemJS 可以在浏览器环境中加载模块。

IIFE

IIFE (Immediately Invoked Function Expression) 是一种立即执行函数表达式，用于创建一个独立的作用域。IIFE 可以用于模块化开发，避免全局变量污染。

UMD

UMD (Universal Module Definition) 是一种通用模块定义规范，兼容 CommonJS、AMD 和全局变量三种模块定义方式。UMD 模块可以在 Node.js 环境和浏览器环境中加载。
