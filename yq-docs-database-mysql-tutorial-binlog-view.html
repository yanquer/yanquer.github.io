Binlog查看

修改于: 2025-11-10
关键字
  • mysql Binlog

在数据库查看 Binlog 日志

mysql -h xxx.com -uroot -p

# 输入密码

# 查看是否开启binlog
show variables like 'log_bin';

# 查看binlog文件列表
show binary logs;

# 查看当前使用的binlog文件
show master status;

# 查看binlog文件内容
# 替换为对应的binlog文件名
show binlog events in 'mysql-bin.000001';

# 查询指定时间类型, 比如删除, 好像不行, 等后面测试一下
show binlog events in 'mysql-bin.000001'
  where event_type = 'Delete_rows_log_event';

Binlog文件是以二进制格式存储的, 不能直接查看. 需要使用 mysqlbinlog 工具来查看.

# 导出binlog文件内容到文本文件
mysqlbinlog -h xxx.com -uroot -p mysql-bin.000001 > binlog.txt

# 查看文本文件内容
cat binlog.txt


#####
# --- 分割线 ---
# 查看删除的部分日志
sudo mysqlbinlog --verbose --base64-output=DECODE-ROWS   /var/lib/mysql/binlog.000018 | grep -A 10 -B 2 "DELETE"


#####
# --- 分割线 ---
# 还原指定的记录到数据库
mysqlbinlog --start-datetime="2025-04-01 00:00:00" --stop-datetime="2025-04-01 10:30:00" /var/lib/mysql/binlog.000001 | mysql -u root -p

说明

Binlog解析后的格式, 一般每一个事务以 BEGIN 开头, 以 COMMIT 结尾.

  • at 后面跟的是时间戳, 可以用 date -d @时间戳 转换为可读格式.
  • # 后面跟的是位点信息, 包括文件名和位置.
  • ### 后面跟的是具体的SQL语句, 包括 INSERT , UPDATE , DELETE 等等.
  • ### SET 后面跟的是一些元信息, 包括数据库名, 表名等.
  • ### WHERE 后面跟的是更新或者删除的条件.
  • ### VALUES 后面跟的是插入的值.
