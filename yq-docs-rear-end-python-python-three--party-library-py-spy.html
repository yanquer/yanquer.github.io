py-spy

修改于: 2025-04-14
相关资源
  • https://pypi.org/project/py-spy/

py-spy 是一个用于 Python 程序的抽样分析器，可以在不修改代码或重启程序的情况下，实时分析 Python 程序的性能。

它可以生成火焰图，帮助开发者识别性能瓶颈。

安装:

pip install py-spy

# rust
# cargo install py-spy
# brew
# brew install py-spy

使用方式

py-spy 可以通过命令行工具使用，也可以作为 Python 库使用。

命令行工具

支持命令

record

记录性能数据

使用 record 命令可以将性能数据记录到文件中，例如生成火焰图:

py-spy record -o profile.svg --pid 12345
# 或者
py-spy record -o profile.svg -- python myprogram.py

这将生成一个交互式的 SVG 文件，展示程序的性能瓶颈。

top

实时查看性能数据

使用 top 命令可以实时查看程序中占用时间最多的函数，类似于 Unix 的 top 命令:

py-spy top --pid 12345
# 或者
py-spy top -- python myprogram.py

这将显示一个实时更新的高层次视图，帮助你快速定位性能问题。

支持按键操作:

1 按％Own排序（当前在该函数中花费的时间的百分比）
2 按％Total排序（函数及其子级中当前的时间百分比）
3 按OwnTime排序（函数中花费的总时间）
4 按TotalTime排序（该函数及其子项花费的总时间）
dump

导出调用堆栈

使用 dump 命令可以显示每个 Python 线程的当前调用堆栈:

py-spy dump --pid 12345

这对于需要单个调用堆栈来确定程序卡住位置的情况非常有用。

gil

显示 GIL 状态

使用 gil 命令可以显示 Python 程序的 GIL 状态:

py-spy gil --pid 12345

这对于分析多线程程序的性能问题非常有用。

其它参数选项

--rate 设置采样频率，默认值为每10毫秒进行一次采样。
--pid 指定要分析的 Python 进程的 PID。

用例

# 获取当前 Python 进程的堆栈信息
py-spy stack --pid 1234

# 获取当前 Python 进程的堆栈信息，并保存为文件
py-spy record --pid 1234 --output stack.svg

# 获取当前 Python 进程的堆栈信息，并保存为文件，并限制堆栈信息的深度为 10
py-spy record --pid 1234 --output stack.svg --depth 10
