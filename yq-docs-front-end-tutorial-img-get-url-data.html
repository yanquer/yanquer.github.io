img标签手动获取url数据

修改于: 2025-01-13

为什么需要这一步, 因为 img 标签的 url 指向的数据获取, 虽然是get接口, 但是也需要token验证, 又因为 img 获取是浏览器行为, 没发自定义, 所以只好自行处理了.

我的场景是网页里嵌套了富文本编辑器, 富文本编辑器是 tinymce. tinymce 的md源码含有的图片上传后无法在前端渲染出来.

此时需要给 tinymce 使用 iframe 增加一个 js 事件来将标签转换一下. 此时可以给 tinymce 的初始化选项加一个这个配置:

// 在 tinymce.init 选项配置中加

init_instance_callback: (editor: any) => {

    // 获取 TinyMCE 渲染的 iframe
    // const iframe: Window = editor.getDoc().defaultView;
    const iframe = editor.iframeElement;
    // const iframeWindow = iframe.contentWindow;
    const iframeDocument = iframe.contentDocument;

    // 添加窗口加载完毕的事件监听器, 没用, 可能是家监听的时候已经加载完成了...
    // iframeWindow.addEventListener('load', () => {
    //     console.log('Iframe content has loaded!');
    //     // 在这里执行你需要的操作
    //     this.convertImgTag(iframeDocument)
    // });
    // iframeDocument.addEventListener('load', () => {
    //     console.log('Iframe content has loaded!');
    //     // 在这里执行你需要的操作
    //     this.convertImgTag(iframeDocument)
    // });

    this.convertImgTag(iframeDocument).then();
},

convertImgTag 实现:

protected convertImgTag = async (doc?: Document): Promise<any> => {
    const imgArray = doc!.getElementsByTagName('img');
    for (let i = 0; i < imgArray.length; i++) {
        const img = imgArray[i]
        const src = img.src
        // 不处理 base64
        if (src.startsWith("data:image/")){

        } else {
            const useUrl = src.startsWith("file://") ? src.replace("file://", "") : src

            // 这个需要自行实现后端接口获取 ArrayBuffer
            //  注意, 如果你用 axios.get 获取的是 utf8 字符串形式的内容,
            //  多半是没有给 get 加config的响应配置, 比如
            //  axios.get("xxx", {
            //    // 设置响应结果为 buffer
            //    responseType: 'arraybuffer'
            //  })
            const srcBuffer: ArrayBuffer = (await this._amApiManager.dispatch<AmReturn>({
                    url: this.url.amisUrl.visitImg,
                    method: "get",
                    data: {url: useUrl} as VisitUrlReq,
                })
            ).data

            const srcBlob = new Blob([srcBuffer], {type: "image/png"})
            const srcBlobUrl = URL.createObjectURL(srcBlob);
            img.src = srcBlobUrl
        }
    }
}

记得这些创建的 blobUrl 最好是在关闭编辑器的时候回收一下:

URL.revokeObjectURL(srcBlobUrl)
