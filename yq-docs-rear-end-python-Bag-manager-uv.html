uv管理器

修改于: 2024-03-12

uv 是一个 Python 的包管理器，类似于 pip，但它提供了更快的安装速度和更好的性能。 支持多版本、多平台、多架构的包管理。

使用方式类似前端的 yarn.

安装

# macOS
brew install uv

# windows

建议使用包管理器全局安装, 否则要指定虚拟环境版本的时候会怪怪的

常用指令

# 非uv环境时初始化
uv init

# 创建一个项目
uv create myproject

# 创建一个包
uv create mypackage

# 运行项目
uv run myproject

# 运行包
uv run mypackage

# 运行包中的文件
uv run mypackage/myfile.py

# 运行包中的文件
uv run mypackage/myfile.py --args "arg1 arg2"

# 安装包并加到 toml 配置
uv add xxx
# 对于一些不是生产环境的包, 加到 dev-dependencies 中
uv add xxx --dev

# 创建项目且指定python版本的虚拟环境
uv create myproject --python 3.8
# 或者在项目已经存在的情况下
uv init --python 3.8
# 当前目录创建指定python版本的虚拟环境, 或者使用路径也可以
uv venv --python 3.8

# 查看系统中可用的Python版本
uv python list

# 查看远程可用的Python版本
uv python list --remote

# 安装Python 3.12
uv python install 3.12

镜像源的配置

参考: https://www.cnblogs.com/etfolinbase/p/18951944

方案一: 环境变量配置

windows可以使用 UV_DEFAULT_INDEX 来配置镜像源地址, 比如:

UV_DEFAULT_INDEX = "https://pypi.tuna.tsinghua.edu.cn/simple"

方案二: 配置文件配置(用户级别)

windows

# %APPDATA%\uv\uv.toml
[[index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true

mac/linux

# ~/.config/uv/uv.toml 或者 /etc/uv/uv.toml
[[index]]
url = "https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple/"
default = true

如果是mac使用brew, 在环境变量配置

export HOMEBREW_PIP_INDEX_URL="https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple"

方案三: 配置文件配置(项目级别)

pyproject.toml 中添加

[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true

或在 uv.toml 中添加：

[[index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true

方案四: 命令行参数配置

可以在命令行中使用 --index-url 参数来指定镜像源地址, 比如:

# uv add requests --index-url https://pypi.tuna.tsinghua.edu.cn/simple
uv add --default-index https://pypi.tuna.tsinghua.edu.cn/simple requests

一些问题

uv即使配置了镜像源, 首次使用的时候, 会发现还是会去 github 下载 CPython, 因为镜像地址没有这个...

可行的解决方式是开梯子, 或者手动下载好这个文件, 然后手动配置环境

windows的可行方案参考: https://zhuanlan.zhihu.com/p/1921615294943724122

# https://zhuanlan.zhihu.com/p/1921615294943724122

# 通过环境变量 UV_PYTHON_INSTALL_MIRROR 构建一个cpython缓存路径
$env:UV_PYTHON_INSTALL_MIRROR='file://D:/IDE/local_python'
# 而这个local_python 文件排布应该是, 有个日志标注文件夹，文件夹内部存放cpython包。
#   需要手动下载好对应版本的cpython包放到该目录下
D:/IDE/local_pythonuv
└─20250612
    └cpython-3.13.5+20250612-x86_64-pc-windows-msvc-install_only_stripped.

设置完环境变量后，就可以创建对应环境了

#
$env:UV_PYTHON_INSTALL_MIRROR='file://D:/IDE/local_python'
uv venv --python 3.13
