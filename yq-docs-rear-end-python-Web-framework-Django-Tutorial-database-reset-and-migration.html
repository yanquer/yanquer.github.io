数据库重置与迁移

修改于: 2025-03-30

正常情况下当模型有变化时, 我们需要更新数据库

python manage.py makemigrations [模型]
# 第一次执行会将建表语句储存在 [模型]/migrations/0001_initial.py
# 可选, 查看对应的sql语句
# python manage.py sqlmigrate [模型] 0001
python manage.py migrate

有时候数据存在问题, 或者需要重置数据库

python manage.py flush

仅 flush , 提示确认删除数据时, 输入 yes 即可

重要

flush是清空数据库所有内容, 但不会删除表结构.

谨慎使用

如果只是清楚某个app的数据, 建议备份后 python manage.py migrate [模型] zero 回滚后再 python manage.py migrate [模型] 应用迁移.

如果只是想清楚某个表的数据, 建议使用SQL语句 DELETE FROM [表名]; 或者 TRUNCATE TABLE [表名]; 区别是 DELETE 会记录日志, TRUNCATE 不会.

或者者使用Django的ORM:

from [模型].models import [模型类]
[模型类].objects.all().delete()

数据库备份恢复

备份:

# python manage.py dumpdata $app > $bak_file.json

# 或者使用 --output 参数指定输出文件, 建议使用 output 来避免 shell 重定向的问题
#   比如丢失引号等
# --indent 参数指定缩进, 方便阅读
python manage.py dumpdata $app --output=$bak_file.json --indent 2

恢复:

python manage.py loaddata $bak_file.json

注解

loaddata 会根据主键进行插入, 将其覆盖. 如果存在外键依赖, 需要保证依赖的表数据已经存在.

如果数据库存在多余的记录, loaddata 不会删除这些记录.

所以最好的恢复备份措施是, 备份的时候全量备份, 恢复的时候先 flush 清空数据库, 然后再 loaddata 恢复数据.

如果是单个表, 在处理好外键等数据库依赖后, 最好是先删除所有数据再 loaddata. 如果新的数据是必要的, 还是直接loaddata即可. .. 可以考虑使用 --natural-primary 和 --natural-foreign 参数来进行恢复.

迁移说明/迁移回滚

查看当前迁移情况

python manage.py showmigrations

输出中存在 [X] 表示已经迁移, [ ] 表示未迁移

一般在 python manage.py makemigrations [模型] 会生成一个 [ ] 的迁移文件, 如果想不应用这个迁移文件, 删除对应的 模型/migrations/迁移名称.py 文件即可,

如果想回滚此应用的所有迁移

python manage.py migrate [模型] zero

# 比如 回滚admin应用的迁移：
# python manage.py migrate admin zero

# 然后应用所有迁移：
python manage.py migrate

如果想回滚到某个迁移文件, 或者说回滚特定应用的最近一次迁移：

python manage.py migrate <app_name> <previous_migration_name>

# 例如，如果要回滚users应用的最近一次迁移：
python manage.py migrate users 0002_previous_migration

高级选项

查看迁移的SQL语句（了解将执行什么操作）：

python manage.py sqlmigrate <app_name> <migration_number>

伪造回滚（只修改迁移记录而不实际修改数据库）：

python manage.py migrate --fake <app_name> <previous_migration_number>

误删迁移文件

有时候可能想当然的认为删除迁移文件, 跟数据库文件就行, 其实不行, 会导致 python manage.py makemigrations [模型] 出现 No changes detected , 无法生成任何内容. 因为实际上, Django 会在数据库中记录已经应用的迁移文件.

解决办法

先做一个空迁移, 生成第一个空的迁移文件

python manage.py makemigrations --empty [模型]

然后再按照正常的流程生成即可

python manage.py makemigrations [模型]
python manage.py migrate
