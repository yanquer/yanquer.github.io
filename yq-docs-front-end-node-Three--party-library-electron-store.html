electron-store

修改于: 2024-07-09
相关资源
  • npm地址: https://www.npmjs.com/package/electron-store
  • github: https://github.com/sindresorhus/electron-store

安装

yarn add electron-store --save

用于数据持久化存储到本地磁盘

数据存储位置(默认是JSON格式)

app.getPath('userData')

支持直接在主进程与渲染进程中使用

使用

import Store from 'electron-store';

const store = new Store();

store.set('unicorn', '🦄');
console.log(store.get('unicorn'));
//=> '🦄'

// Use dot-notation to access nested properties
store.set('foo.bar', true);
console.log(store.get('foo'));
//=> {bar: true}

store.delete('unicorn');
console.log(store.get('unicorn'));
//=> undefined

支持直接定义数据格式

import Store from 'electron-store';

const schema = {
  foo: {
    type: 'number',
    maximum: 100,
    minimum: 1,
    default: 50
  },
  bar: {
    type: 'string',
    format: 'url'
  }
};

const store = new Store({schema});

console.log(store.get('foo'));
//=> 50

store.set('foo', '1');
// [Error: Config schema violation: `foo` should be number]

使用YAML格式

import Store from 'electron-store';
import yaml from 'js-yaml';

const store = new Store({
  fileExtension: 'yaml',
  serialize: yaml.safeDump,
  deserialize: yaml.safeLoad
});

渲染进程中使用

如果单独分离在两个进程中使用, 是可行的, 但是这样一来数据就不是互通的了.

如果需要两者数据互通, 建议的方式:

主进程

ipcMain.handle('getStoreValue', (event, key) => {
  return store.get(key);
});

渲染进程

const foo = await ipcRenderer.invoke('getStoreValue', 'foo');
