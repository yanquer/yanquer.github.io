Appium是一个开源的自动化测试工具，用于测试移动应用程序，支持iOS和Android平台。

注解

支持不同平台实际是依赖于不同的驱动器，比如 uiautomator2 驱动器用于Android平台。

不过目前主流驱动器仅支持各个平台的原生应用或Web应用, 不支持混合开发的, 比如在安卓上就没 通用的支持安卓游戏(基于C开发)的驱动器.

建议全局安装:

npm i -g appium

驱动

编写不同平台的测试脚本时，需要使用不同的驱动器。

安装官方驱动:

appium driver install uiautomator2

驱动管理:

# 从npm安装官方驱动
appium driver install <driver-name>
# Install any driver from npm
appium driver install --source=npm <driver-name>
# See documentation for installation from other sources

# 列出所有已安装的驱动
appium driver list --installed
# 更新一个已经安装的驱动
# 这将不会更新主要版本，以防止破坏性更改
appium driver update <driver-name>
# 更新一个已经安装的驱动到最新版本(可能包含破坏性更改)
appium driver update <driver-name> --unsafe
# 卸载一个驱动(它不会永久存在，对吧?)
appium driver uninstall <driver-name>

客户端(可选)

当你想用其它语言编写测试脚本时，你可以通过使用Appium客户端来解决

Python

当 启动 后, 可以通过以下命令安装Python客户端:

pip install Appium-Python-Client

这个示例使用Python内置的 unittest 模块， 尽管你可以使用任何你想要的Python测试框架。 Appium Python客户端自动添加 appium: 供应商前缀。 通常你不需要担心前缀。

import unittest
from appium import webdriver
from appium.options.android import UiAutomator2Options
from appium.webdriver.common.appiumby import AppiumBy

capabilities = dict(
    platformName='Android',
    automationName='uiautomator2',
    deviceName='Android',
    appPackage='com.android.settings',
    appActivity='.Settings',
    language='en',
    locale='US'
)

appium_server_url = 'http://localhost:4723'

class TestAppium(unittest.TestCase):
    def setUp(self) -> None:
        self.driver = webdriver.Remote(appium_server_url, options=UiAutomator2Options().load_capabilities(capabilities))

    def tearDown(self) -> None:
        if self.driver:
            self.driver.quit()

    def test_find_battery(self) -> None:
        el = self.driver.find_element(by=AppiumBy.XPATH, value='//*[@text="Battery"]')
        el.click()

if __name__ == '__main__':
    unittest.main()

capabilities配置项说明

即上面 webdriver.Remote(appium_server_url, options=UiAutomator2Options().load_capabilities(capabilities)) 的 options 配置参数

  • platformName: 平台名称

  • automationName: 自动化引擎名称

  • deviceName: 设备名称, 使用 adb connect ip:port 连接后, 用 ip:port 的形式或者可以使用 adb devices -l 查看, 设备名称就在输出的 product: 后面.

    比如: 'deviceName': '127.0.0.1:7555',, 'deviceName': 'mumu',

  • appPackage: 应用包名

  • appActivity: 应用Activity

  • unicodeKeyboard: 使用自带输入法，输入中文时填True

  • resetKeyboard: 执行完程序恢复原来输入法

  • noReset: 是否重置应用状态.

    是否要重置App，如果为False的话，执行完脚本后，app的数据会清空，比如你原本登录了，执行完脚本后就退出登录了

  • newCommandTimeout: 会话超时时间

  • automationName: 自动化引擎名称

  • uiautomator2ServerInstallTimeout: 安装uiautomator2服务的超时时间

  • uiautomator2ServerLaunchTimeout: 启动uiautomator2服务的超时时间

  • adbExecTimeout: adb执行超时时间

查看appPackage和appActivity

方案一

在模拟器中, 打开应用后, 查看日志

# 先打开日志
adb logcat | findstr "cmp"

然后手动点击启动应用. 看看日志最新增加的一行 cmp 信息, 比如 乱斗西游2:

05-22 20:56:02.734  1175  1523 I ActivityTaskManager: START u0 {act=android.intent.action.MAIN cat=[android.intent.category.LAUNCHER] flg=0x10200000 cmp=com.netease.ldxy.mi/com.netease.ntunisdk.external.protocol.ProtocolLauncher bnds=[180,925][620,1260]} from uid 1000, pid 1477
05-22 20:56:03.398  1175  2115 I ActivityTaskManager: START u0 {flg=0x24000000 cmp=com.netease.ldxy.mi/com.netease.ldxy.Launcher bnds=[180,925][620,1260]} from uid 10035, pid 13536

包名+Activity启动名为:

com.netease.ldxy.mi/com.netease.ntunisdk.external.protocol.ProtocolLauncher

下次就用这个信息启动 乱斗西游2

adb shell am start -n com.netease.ldxy.mi/com.netease.ntunisdk.external.protocol.ProtocolLauncher
方案二

启动需要查询的APP后(这里是bilibili)查询活动日志

adb shell dumpsys activity recents | find "intent={"

# 比如输出为
#    intent={act=android.intent.action.MAIN cat=[android.intent.category.LAUNCHER] flg=0x10200000 cmp=tv.danmaku.bili/.ui.splash.SplashActivity}
#
# cmp的值：tv.danmaku.bili/.ui.splash.SplashActivity，
#   /前的 tv.danmaku.bili 就是 appPackge 的值，
#   /后的 .ui.splash.SplashActivity 就是 appActivity 的值。

元素定位

参考: CSDN-appium AppiumBy定位方法【一】

from appium.webdriver.common.appiumby import AppiumBy

self.driver.find_element(by=AppiumBy.XPATH, value='//*[@text="接受"]')

XPATH定位

确定元素的情况

根据ID属性定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[@resource-id="com.android.settings:id/search"]')

根据文本定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[@text="Battery"]')

class属性定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[@class="android.widget.TextView"]')

content-desc属性定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[@content-desc="Search"]')
不确定元素内容-模糊定位

ID属性模糊定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[contains(@resource-id, "search")]')

根据文本模糊定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[contains(@text, "Bat")]')

class属性模糊定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[contains(@class, "TextView")]')

content-desc属性模糊定位元素:

self.driver.find_element(by=AppiumBy.XPATH, value='//*[contains(@content-desc, "Sear")]')

元素操作

# 点击
el.click()

# 输入
el.send_keys('hello')

# 获取文本
el.text

# 获取属性
el.get_attribute('resource-id')

# 获取坐标
el.location

# 获取大小
el.size

# 获取标签名
el.tag_name

# 获取元素是否可见
el.is_displayed()

# 获取元素是否启用
el.is_enabled()

# 获取元素是否被选中
el.is_selected

插件(可选)

插件是一种扩展Appium的方式，可以添加新的功能或者修改现有功能。

使用方式:

appium --use-plugins=<plugin-name>

插件管理:

# Install an official plugin from npm (see documentation for a list of such plugins)
appium plugin install <plugin-name>
# Install any plugin from npm
appium plugin install --source=npm <plugin-name>
# See documentation for installation from other sources

# List already installed plugins
appium plugin list --installed
# Update a plugin (it must be already installed)
# This will NOT update the major version, in order to prevent breaking changes
appium plugin update <plugin-name>
# Update a plugin to the most recent version (may include breaking changes)
appium plugin update <plugin-name> --unsafe
# Uninstall a plugin
appium plugin uninstall <plugin-name>

官方文档插件: http://appium.io/docs/en/latest/ecosystem/plugins/

启动

启动Appium服务:

# Start the server on the default host (0.0.0.0) and port (4723)
appium server
# You can also omit the 'server' subcommand
appium
# Start the server on the given host, port and use a custom base path prefix (the default prefix is '/')
appium --address 127.0.0.1 --port 9000 --base-path /wd/hub

链接安卓模拟器

此处以MuMu模拟器为例. 自己去模拟器官方查询一下提供的adb端口是多少, 比如MuMu模拟器的默认端口是7555. 那么使用adb就链接这个端口.

# Windows下, 如果有配置自己的Android SDK环境变量, 可以直接使用adb命令
#   或者去Android SDK的platform-tools目录下使用adb命令
# 如果没有配置过, 可以使用 MuMu模拟器自带的adb命令, 一般在MuMu模拟器的安装目录下的shell目录下
#   比如: D:\Program Files\MuMu\emulator\MuMuPlayer-12.0\shell

# 如果之前打开了, 需要先关闭服务
# adb kill server

# 启动服务并连接连接设备
adb connect 127.0.0.1:7555
# 此时就可以在 appuim 配置 deviceName 为 127.0.0.1:7555 链接使用了

# 查看设备列表以及设备名, deviceName 也可以用这里的名字 (product: 后面的名字)
adb devices -l
