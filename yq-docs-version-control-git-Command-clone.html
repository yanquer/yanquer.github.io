clone

修改于: 2024-06-25

将远程仓库克隆到本地

选项参数

参考: https://stackoverflow.com/questions/3959924/whats-the-difference-between-git-clone-mirror-and-git-clone-bare

--bare

将获得复制的所有标签(tags)，本地分支master（HEAD），next，pu和maint，没有远程跟踪分支。

也就是说，所有分支都是按原样复制的，并且它完全独立设置，没有期望再次获取。任何远程分支（在克隆的远程中）和其他引用都被完全忽略。

--mirror

这些引用中的每一个都将按原样复制。

您将获得所有标签、本地分支 master (HEAD)、next、pu 和 maint、远程分支 devA/master 和 devB/master、其他引用 refs/foo/bar 和 refs/foo/baz。

一切都与克隆的远程完全相同。远程跟踪设置为，如果您运行 git remote update，所有引用都将从原点覆盖，就像您刚刚删除镜像并重新克隆它一样。 正如文档最初所说，它是一个镜像。它应该是功能相同的副本，可以与原始副本互换。

下面两种是等价的

$ git clone --mirror $URL

# 相当于
$ git clone --bare $URL
$ (cd $(basename $URL) && git remote add --mirror=fetch origin $URL)

bare,mirror,clone对比

git clone's config file looks like:

[remote "origin"]
    url = https://github.com/example
    fetch = +refs/heads/*:refs/remotes/origin/*

git clone --bare's config file looks like:

[remote "origin"]
    url = https://github.com/example

git clone --mirror's config file looks like:

[remote "origin"]
    url = https://github.com/example
    fetch = +refs/*:refs/*
    mirror = true

实例: --bare 裸库使用

参考: https://stackoverflow.com/questions/3959924/whats-the-difference-between-git-clone-mirror-and-git-clone-bare

与正常库区别:
  • 正常库/普通库：使用git init创建即可。包含了工作区，可以正常的进行源文件的编写，提交等各种git常规操作。
  • 裸库：使用git init --bare创建。它主要是只保存历史提交的版本信息，不保存文件。作用就是作为分享库。

下面以百度的amis仓库为例(自行测试可选其他库, 这库有点大):

https://github.com/baidu/amis.git

克隆裸库

# --recursive可选, 适用于需要子仓库一起克隆的(正常库是, 这里不知道)
git clone --bare --recursive https://github.com/baidu/amis.git


# 如果要创建一个新的裸库,
# cd /test-bare-repo
# git init --bare

更新库

# 从 origin 覆盖
git remote update

# 这个不确定是不是...
# 删除已经不存在于上游的远程分支引用
# git fetch --prune

其他可用参数

# 查看当前头hash
git rev-parse HEAD
d4c540e121a664b7191ed3ac2e6c6128055cfb37

# 切换当前指向的分支
git symbolic-ref HEAD refs/heads/pre-release
