clone

修改于: 2024-06-12

将远程仓库克隆到本地

选项参数

参考: Git: git clone –mirror和git clone –bare之间的区别

-–mirror

克隆一个裸露（bare）镜像，这个镜像包含了 原始存储库的所有数据、引用和设置。 镜像存储库是一个完全复制的存储库，可以用于创建一个原始存储库的镜像副本。这意味着如果对镜像仓库进行更改，那么原始仓库也会受到影响。

让我们看一个示例，假设我们有一个名为original_repo的存储库，并使用以下命令创建一个镜像副本：

git clone --mirror original_repo mirror_repo

现在，如果我们在mirror_repo中添加、修改或删除分支、标签或提交，那么这些更改也会反映在original_repo中。

适用于需要创建一个原始仓库的完全镜像副本的情况

源于AI

git clone --mirror 命令会创建一个镜像仓库,并自动设置好远程仓库的 fetch 和 push 配置。 但是要实际同步更新,仍然需要手动执行 git fetch --prune 命令。这个命令会拉取远程所有更新,并删除本地已经不存在的分支和标签。 如果需要将镜像仓库推送到其他地方,也需要手动执行 git push --mirror 命令。这样可以将镜像仓库的所有更新同步到其他地方。
-–bare

与git clone --mirror命令相比，git clone --bare命令会克隆一个裸露（bare）存储库。 裸露存储库只包含 原始存储库的版本历史 ，而不会包含分支、标签或其他引用。它适用于创建一个干净的、不包含任何引用的存储库。裸露存储库一般用于共享和备份的目的。

让我们通过一个示例来进一步理解。假设我们有一个名为original_repo的存储库，并使用以下命令创建一个裸露克隆：

git clone --bare original_repo bare_repo

在裸露存储库中，我们只能看到提交历史，而分支和标签是不可见的。这使得裸露存储库成为一个紧凑且适用于共享和备份的选项。

适用于需要一个干净的存储库的情况，用于共享和备份目的，而不需要分支和标签的内容。
