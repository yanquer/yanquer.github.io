集群安装

k8s集群主流安装方式对比分析

  • minikube
  • 二进制安装
  • kubeadm等安装工具

核心组件

静态Pod的方式:

## etcd, apiserver, controller-manager, kube-scheduler
$ kubectl -n kube-system get po

systemd服务方式:

$ systemctl status kubelet

k8s安装

安装

# macOS
brew install kubectl

# windows
choco install kubernetes-cli

# linux - debian - amd64
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

# 查看版本
kubectl version --client

本地集群安装配置

minikube参考: https://minikube.sigs.k8s.io/docs/start/

一般来说, 本地就自己一个机器, 手动一个一个去安装虚拟机测试也比较麻烦, 这时可以使用 minikube 来快速搭建一个本地的 k8s 集群。

安装

# macOS
brew install minikube

# windows
choco install minikube

# linux - debian
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube_latest_amd64.deb
sudo dpkg -i minikube_latest_amd64.deb

minikube常用指令

# 启动
minikube start

# 停止
minikube stop

# 删除
minikube delete --all

# dashboard, 不加 --url 会自动打开浏览器
minikube dashboard --url

# 查看集群状态
minikube status

常用指令

常见子指令格式

kubectl get       # - 列出资源
kubectl describe  # - 显示有关资源的详细信息
kubectl logs      # - 打印 Pod 中容器的日志
kubectl exec      # - 在 Pod 中的容器上执行命令

一些实例

# 获取所有服务
kubectl get svc --all-namespaces

# 获取指定命名空间的所有pod
kubectl get pods -n cert-manager

# 通过yaml文件创建资源
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v1.13.0/deploy/static/provider/aws/deploy.yaml
# 等价于使用 heml 包管理器 安装

# 获取集群信息
kubectl cluster-info

# 获取部署资源信息
kubectl get deployments

# 获取节点信息
kubectl get nodes

# 打开一个代理终端, 以便访问被隔离的容器
kubectl proxy
