反向查询related_name和related_query_name

修改于: 2024-12-25

参考: https://blog.csdn.net/coolzpf/article/details/132844054

related_name 和 related_query_name 是 Django ORM 中用于定义模型之间关系的两个重要参数, 它们主要用于反向查询关联模型对象.

比如有一个这样的应用

class Question(models.Model):
  name = models.CharField('问题名称', max_length=64)

class Answer(models.Model):
    question = models.ForeignKey(Question, on_delete=models.CASCADE,
                                  verbose_name='关联的问题')
    content = models.TextField('答案的内容')
related_name

用于反向查询模型对象本身;

默认值为 answer_set, 即模型名小写 + _set; 特殊情况, 如果是 OneToOneField 字段, 默认值直接是 模型名小写, 如 answer

用于反向查询 Question 对应的 Answer, 比如 q=Question.objects.all().first(); q.answer_set.all() 表示查询问题一的所有答案. (此处是外键, 多对一关系)

related_query_name

用于反向查询模型对象的字段(跨表查询);

默认值为 answer, 即模型名小写.

同样用于反向查询 Question 对应的 Answer, 不过是具体到字段查询时候的名字, 比如 Question.objects.all().filter(answer__content="自主学习") 表示查询答案内容为 "自主学习" 的问题.

当 related_name 有手动设置时, related_query_name 会自动设置为 related_name 的值.

反向查询中 _set 的使用规则

关系类型 无 related_name 有 related_name 命名约定
ForeignKey 使用 模型名小写_set 使用 related_name 值 使用复数形式 (answers)
ManyToManyField 使用 模型名小写_set 使用 related_name 值 使用复数形式 (answers)
OneToOneField 直接使用 模型名小写 使用 related_name 值 使用单数形式 (answer)

示例

一对一字段(OneToOneField)

# models.py
class UserProfile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    bio = models.TextField()

# 反向查询 - 通过小写模型名
user = User.objects.get(id=1)
profile = user.userprofile  # 直接访问

多对一字段(ForeignKey)

# models.py
class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    author = models.ForeignKey(Author, on_delete=models.CASCADE)
    title = models.CharField(max_length=200)

# 反向查询 - 使用 related_name 或 默认 _set
author = Author.objects.get(id=1)
books = author.book_set.all()  # 使用默认的 _set

多对多字段(ManyToManyField)

# models.py
class Student(models.Model):
    name = models.CharField(max_length=100)

class Course(models.Model):
    students = models.ManyToManyField(Student)
    title = models.CharField(max_length=200)

# 反向查询 - 使用 related_name 或 默认 _set
student = Student.objects.get(id=1)
courses = student.course_set.all()  # 使用默认的 _set

使用 related_name 和 related_query_name 的例子

class Question(models.Model):
    name = models.CharField('问题名称', max_length=64)

class Answer(models.Model):
    question = models.ForeignKey(Question, on_delete=models.CASCADE,
                                  verbose_name='关联的问题',
                                  related_name='answers',
                                  related_query_name='answer')
    content = models.TextField('答案的内容')

# 反向查询 - 使用 related_name
question = Question.objects.get(id=1)
answers = question.answers.all()  # 使用 related_name

# 反向查询 - 使用 related_query_name, 跨表字段查询
questions = Question.objects.filter(answer__content="自主学习")  # 使用 related_query_name
