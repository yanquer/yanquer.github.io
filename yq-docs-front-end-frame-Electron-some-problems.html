一些问题

修改于: 2023-02-26

preload无法导入其他ts

BrowserWindow加载参数webPreferences.preload,

当preload调用了node的api时, 需要将 nodeIntegration 设置为 true:

const mainWindow = new BrowserWindow({
      width: 800,
      height: 600,
      webPreferences: {
          // 集成nodejs, preload脚本中使用了Node.js的API时必设为true, 才能正常执行
          nodeIntegration: true,
          preload: join(__dirname, 'preload.js')
      }
  })

静态资源处理(MacOS)

参考
  • Electron常见问题（三）常用路径/静态资源丢失
  • electron的extraFiles和extraResources的配置和使用（附：动态路径拼接问题）

关键字: Electron静态资源

默认情况下, resources 目录下图标等资源不会直接全部复制到生成目录下, 需要显示配置哪些资源是静态资源, 比如在 electron-builder.yml 配置 (不指定别名可以单独一个 - resources)

extraResources:
#  - resources
  - from: resources
    to: resources

就会将当前目录下的 resources 文件见复制到生成目录下, MacOS下是 生成APP/Contents/Resources 下面

图标问题(MacOS)

参考
  • Electron托盘图标显示问题

关键字: Electron菜单栏/任务栏图标

其实还是静态资源问题, 通过 静态资源处理(MacOS)_ 即可解决.

另外需要注意的一点是, 菜单栏的图标, 不能使用 icns 文件, 只能使用普通的图片, 比如 png (这一点几乎没看到有地方提到), 且需要使用 nativeImage.createFromPath 转换为 nativeImage

import {Tray, nativeImage} from 'electron';

tray = new Tray(nativeImage.createFromPath(menuImg));

electron打包后可能出现的问题

启动白屏

说明渲染进程没有正常启动, 注意几点

  • 打包后的正式版, 访问地址是以 app: 开头的, 比如 app://./home 而不是 http://localhost: 8080/home
  • 确定构建时, 渲染进程的源码正常输出到配置的输出目录下, 且 electron打包 时正确处理(一般主/渲染进程源码都是输出到同一个目录)

ERR_FILE_NOT_FOUND (-6) loading 'app://./home'

同上, 不过注意第二点

"Could not find ffmpeg executable" using 'app.asar.unpacked'

这个主要是发生在项目有使用 ffmpeg 执行文件的情况, 默认情况下现状都是打包到 app.asar 集成, 比如安装的 ffmpeg-static 就会到 app.asar/node_modules/ffmpeg-static/ffmpeg 这种.

需要在源码里兼容一下路径处理

import ffmpeg from 'ffmpeg-static';

const truthFfmpegPath = process.env.NODE_ENV === 'production' ?
    ffmpeg.replace('app.asar', 'app.asar.unpacked') :
    ffmpeg;

注解

本来有个 electron-util 提供了路径处理方案, 但是不知道为什么笔者使用无效

const util = require('electron-util');
const ffmpegPath = util.fixPathForAsarUnpack(ffmpeg);

参考: https://github.com/kribblo/node-ffmpeg-installer/issues/26

暂未分类报错总结

UnhandledPromiseRejectionWarning: Error: Failed to load image from path

发生于使用 MacOS菜单栏/Windows系统托盘栏 图标时, 菜单栏图标无法成功加载.

此处主要针对MacOS,

解决, 图标不能直接给路径, 要转换为image

menuImg => new Tray(nativeImage.createFromPath(menuImg));

另外

  • 图片的大小需要控制在 16 x 16
  • 图片 menuImg 最好使用 path 获取绝对路径. 相对路径不容易确定当前路径...
