关联说明
  • 还有一个 -webkit-clip-path 属性，用于 WebKit 内核浏览器，但是已经被 clip-path 属性替代。

clip-path 属性使用裁剪方式创建元素的可显示区域。区域内的部分显示，区域外的隐藏。

它可以用来隐藏元素的一部分内容或者创建非矩形的可视区域。 它可以创建各种形状的裁剪区域，如圆形、矩形、三角形、圆角矩形、椭圆、多边形等。 它使用 CSS 的路径语法来定义裁剪区域，可以使用直线、曲线、弧线、贝塞尔曲线等路径命令来创建复杂的形状。

语法

/* Keyword values */
clip-path: none;

/* <clip-source> values */
clip-path: url(resources.svg#c1);

/* <geometry-box> values */
clip-path: margin-box;
clip-path: border-box;
clip-path: padding-box;
clip-path: content-box;
clip-path: fill-box;
clip-path: stroke-box;
clip-path: view-box;

/* <basic-shape> values */
clip-path: inset(100px 50px);
clip-path: circle(50px at 0 100px);
clip-path: ellipse(50px 60px at 10% 20%);
clip-path: polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%);
clip-path: path(
  "M0.5,1 C0.5,1,0,0.7,0,0.3 A0.25,0.25,1,1,1,0.5,0.3 A0.25,0.25,1,1,1,1,0.3 C1,0.7,0.5,1,0.5,1 Z"
);

/* Box and shape values combined */
clip-path: padding-box circle(50px at 0 100px);

/* Global values */
clip-path: inherit;
clip-path: initial;
clip-path: revert;
clip-path: revert-layer;
clip-path: unset;

<clip-source>

none
默认值。元素的可见区域是一个矩形，不进行裁剪。
url()
引用一个 SVG 文件，SVG 文件中定义了一个裁剪路径。

支持的形状<basic-shape>值

inset()

定义一个 inset 矩形, 基础用法类似于 margin 属性

如下例子定义了一个矩形的裁剪区域：

clip-path: inset(20px 40px 60px 80px round 15px);
clip-path: inset(0 round 15px);
  • 上右下左边距: 分别指定上、右、下、左边界的偏移量，可以是长度值或百分比。
  • round: 可选，用于设置圆角。后面跟一个或四个长度值，分别对应全部圆角或按顺序对应每个角的圆角半径。
circle()

定义一个圆形（使用一个半径和一个圆心位置）。

如下例子定义了一个圆形的裁剪区域：

clip-path: circle(50% at 50% 50%);
  • 半径: 50% 指定圆的半径。
  • 位置: at 50% 50% 定义了圆心的位置。
ellipse()

定义一个椭圆（使用两个半径和一个圆心位置）。

如下例子定义了一个椭圆形的裁剪区域：

clip-path: ellipse(50% 25% at 50% 50%);
  • 水平半径: 50% 水平方向上的椭圆半径。
  • 垂直半径: 25% 垂直方向上的椭圆半径。
  • 位置: 同 circle() 函数，定义了椭圆中心的位置。
polygon()

定义一个多边形（使用一个 SVG 填充规则和一组顶点）。 (随机点)

如下例子定义了一个多边形裁剪区域：

clip-path: polygon(25% 0%, 100% 25%, 75% 100%, 0% 75%);
  • 顶点坐标: 列表形式给出多边形各顶点的坐标，每一对值代表一个点的水平位置和垂直位置。 例如，25% 0% 表示第一个顶点位于元素宽度的25%处，垂直顶部。
path()

定义一个任意形状（使用一个可选的 SVG 填充规则和一个 SVG 路径定义）。

如下例子定义了一个路径裁剪区域

<svg>
  <defs>
    <clipPath id="customClip">
      <path d="M10 10 H90 V90 H10 L10 10 Z M30 30 H70 V70 H30 Z"/>
    </clipPath>
  </defs>
</svg>

<div class="clipped-content" style="clip-path: url(#customClip);"></div>
  • d: SVG 路径数据，定义了裁剪区域的形状。
  • 包含内外两个矩形的剪切区域，外部大矩形剪出基本形状，内部小矩形进一步剪去中间部分。
  • 然后在HTML元素中通过CSS引用这个SVG剪切路径ID。

<geometry-box>

如果同 <basic-shape> 一起声明，它将为基本形状提供相应的参考框盒。 通过自定义，它将利用确定的盒子边缘包括任何形状边角（比如说，被 border-radius 定义的剪切路径）。

几何框盒可以有以下的值中的一个：

margin-box
使用 margin box 作为引用框。
border-box
使用 border box 作为引用框。
padding-box
使用 padding box 作为引用框。
content-box
使用 content box 作为引用框。
fill-box
利用对象边界框（object bounding box）作为引用框。
stroke-box
使用笔触边界框（stroke bounding box）作为引用框。
view-box
使用最近的 SVG 视口（viewport）作为引用框。 如果 viewBox 属性被指定来为元素创建 SVG 视口， 引用框将会被定位在坐标系的原点，引用框位于由 viewBox 属性建立的坐标系的原点，引用框的尺寸用来设置 viewBox 属性的宽高值。
none
不创建剪切路径。
