关于 #!

表示当作为执行文件的时候，使用解释器路径，默认为当前登录shell

必须写在第一行才有用

当执行执行文件时，cd到所在目录后，执行需要加 ./ ， ./test的./是为了承接现在所在的文件夹， 让现在所在的文件夹+文件，合并成该文件的完整路径，用于执行。

优先级
当直接指定解释器的时候 比如 python xxx.py， xxx.py文件里的 #! 是不生效的，

只有作为执行文件的时候才会生效，比如 ./xxx.py
