多对多字段的更新

修改于: 2025-03-28

参考: https://deepinout.com/django/django-questions/48_django_django_direct_assignment_to_the_forward_side_of_a_manytomany_set_is_prohibited_use_userset_instead.html

即更新 models.ManyToManyField 定义的字段.

一般来说, 多对多字段查询形式虽然看起来是个数组, 但是并不能直接将一个数组赋值给它, 需要使用 set() 方法.

比如定义如下模型

from django.db import models

class Blog(models.Model):
    title = models.CharField(max_length=100)
    content = models.TextField()
    tags = models.ManyToManyField('Tag')

class Tag(models.Model):
    name = models.CharField(max_length=50)

错误的更新形式

blog.tags = [tag1, tag2, tag3]

会遇到报错:

Direct assignment to the forward side of a many-to-many set is prohibited. Use singer.set() instead.

正确的更新形式

blog.tags.set([tag1, tag2, tag3])

除了 set() 方法，Django还提供了其他一些用于操作多对多关系的方法：

  • add()：添加一个或多个对象到多对多关系中。
  • remove()：从多对多关系中移除一个或多个对象。
  • clear()：清除多对多关系中的所有对象。
  • set()：设置多对多关系中的对象集合，等同于清除原有关系并添加新的关系。

下面是使用这些方法的示例代码：

# 添加一个标签到博文的多对多关系中
blog.tags.add(tag4)

# 从博文的多对多关系中移除一个标签
blog.tags.remove(tag1)

# 清除博文的多对多关系中的所有标签
blog.tags.clear()

# 设置博文的多对多关系为[tag1, tag2, tag3]
blog.tags.set([tag1, tag2, tag3])

多对多字段查询

比如有一个模型

class Article(models.Model):
    title = models.CharField(max_length=200)
    tags = models.ManyToManyField('Tag')

class Tag(models.Model):
    name = models.CharField(max_length=50)
    category = models.CharField(max_length=50)

查询示例

直接用跨表name查询

# 查询所有包含特定标签的文章
articles_with_tag = Article.objects.filter(tags__name='Django')

# 查询包含多个标签的文章
articles_with_tags = Article.objects.filter(tags__name__in=['Django', 'Python']).distinct()

# 查询标签类别为特定值的文章
articles_with_tag_category = Article.objects.filter(tags__category='Programming')

使用主键或者模型查询

# 查询包含 tag ID 为 1,2,3 中任意一个的文章
articles = Article.objects.filter(tags__in=[1, 2, 3]).distinct()

# 查询必须同时包含 tag 1,2,3 的文章
articles = Article.objects.filter(tags=1).filter(tags=2).filter(tags=3)

# 查询 tag 名称为 "python" 或 "javascript" 的文章
articles = Article.objects.filter(tags__name__in=['python', 'javascript'])

# 查询 tag 名称为 "python" 且分类为 "programming" 的文章
articles = Article.objects.filter(
    tags__name='python',
    tags__category='programming'
)
