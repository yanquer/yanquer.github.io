数据域问题总结

修改于: 2024-10-15

不能解析同层数据域

比如有一个page长这样

{
  "type": "page",
  "id": "testTableId",
  "data": {
      "tItems": "TestTableDataList",
      "tItems2": "${tItems}"
  },
}

数据域处理时, 试图直接用 tItems2 来引用本层数据域的 tItems , 结果是失败的, 只能在下面再开一层数据域

注解

使用 service 时不会新开一个数据域, 还是在当前层的数据域操作. 这一点只是使用的时候感觉到, 官方并没有提出, 待确定

data的作用

data只能提供 初始数据源

没法实现联动变化的效果, 要联动就自行加辅助组件处理.

辅助组件一个问题

关键字
  • 辅助组件
  • 联动更新
  • 值的实时更新
  • 落后更新
  • 落后一个版本更新
  • 延迟更新
  • 延迟一个版本更新

当 value 的值是表达式时, 此组件的值, 有时候是其他组件有引用这个组件的值不会实时更新, 会慢一个版本. strictMode 无效, 此时再包一层 service 解析可以解决.

{
  "type": "container",
  "body": [
      // 辅助组件放在 service 组件来保持实时更新.
      {
          "type": "service",
          "body": [
              {"type": "input-text", "name": "comboConditions1", "label": "辅助组件comboConditions1",
                  "visible": true,
                  "strictMode": false,
                  // 在同层的话 value 的值更新会延迟一个版本... 只有放到 service 下面才会实时联动更新
                  "value": "${comboConditions0|rmKeyAndType:type:data-of-type}", },
          ],
      },
      {
          "type": "page",
          "data": {
              "comboConditions0": "${comboConditions1|addKeyAndType:type:data-of-type}",
          },
          "body": [
              data2,
          ],
      },
  ]
}

或者使用 setValue 的动作来更新. 这种情况更适用于dialog关闭的时候, 点击确认时候的更新上层组件的值.

table-view单项为list时报错

大致代码

{"tds": [{
  "body": [
    {
        "type": "list",
        "source": "${viewData.options.includes|strToListArray}",
        "listItem": {
            "subTitle": "$desc"
        },
    },
  ]
}]}

报错内容:

[mobx] Encountered an uncaught exception that was thrown by a reaction or observer component, in: 'Reaction[ListRenderer.render()]' Error: Maximum update depth exceeded. This can happen when a component repeatedly calls setState inside componentWillUpdate or componentDidUpdate. React limits the number of nested updates to prevent infinite loops.

没有看源代码, 根据目前情况判断, 大致原因是 source 的数据源不是组件, 故没法监听到状态. 实际上估计是库的 bug

临时解决方案: 使用辅助组件.

{"tds": [{
  "body": [
    {
        // "type": "tpl",
        "type": "input-text",
        // "tpl": "${viewData.options.includes|strToListArray}",
        "value": "${viewData.options.includes|strToListArray}",
        "name": "tmpTpl",
        "visible": false,
    },
    {
        "type": "list",
        "source": "$tmpTpl",
        "listItem": {
            "subTitle": "$desc"
        },
    },
  ]
}]}

注解

tpl模版组件不能获取值... 所以还得用 input-text

提出在 https://github.com/baidu/amis/issues/11054

数据域提交时Key的展开

默认情况下, 数据域的数据如果包含 . 或 对象取值, 在提交时候默认会将这些给展开为对象.

用官网给的例子说明

默认请求数据体中配置 key，如果带路径会自动转成对象, 如:

"api": {
    "method": "post",
    "url": "/api/mock2/form/saveForm",
    "data": {
        "a.b": "${name}",
        "c[d]": "${name}"
    }

}

最终发送出去的数据格式为

{
  "a": {
    "b": "xxx"
  },
  "c": {
    "d": "xxx"
  }
}

如果数据映射中的 key 不想被处理路径则需要配置 convertKeyToPath 为 false 如:

"api": {
    "method": "post",
    "url": "/api/mock2/form/saveForm",
    "convertKeyToPath": false,
    "data": {
        "a.b": "${name}",
        "c[d]": "${name}"
    }

}

这样发送的数据格式为

{
  "a.b": "xxx",
  "c[d]": "xxx"
}

即在请求域配置 "convertKeyToPath": false
