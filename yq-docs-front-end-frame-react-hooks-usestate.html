useState

修改于: 2024-03-07

为组件添加 State

例子:

import React, { useState } from 'react';

function Example() {
  // 声明一个state count
  const [count, setCount] = useState(0);

  // 使用count进行渲染，并绑定点击时间，使count+1
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}
  • setCount 的触发是异步的

注解

当有多个state时，使用多个state变量，而不是一个, 这样把无关逻辑分离，便于增删改state

同时, 如果因为UI界面的更新, 需要做一些操作, 同时这个操作需要用到state变量, 那么建议 将此操作放到 useEffect 中, 避免不能获取最新的state值.

另外, 貌似, 在类中的 state 是一直存在的, 不会因为触发而清空, 函数组件中不确定.... 后面有空测试一下

state异步的实时属性访问

比如我有一个计时器, 需要每隔一秒将时间加一

const [timer, setTimer] = useState(0);

useEffect(() => {

  const timerIn = setInterval(() => {
      setTimer(timer+1)
  }, 1000)

  return () => clearInterval(timerIn)

}, [])

这样会发现, 定时器的timer只会增加到一, 因为后续访问的时候, 异步导致拿到的timer一直为 0, 如果需要同步访问值, 需要更新为回调的形式

setTimer((prevTimer) => {
  console.log('timer 已更新为:', prevTimer + 1);
  return prevTimer + 1;
});

状态更新失效-数组/对象

当你有一个状态是数组/对象时, 更新状态不能直接修改数组/对象的某个值, 而是需要重新创建一个新的数组/对象

错误的更新方式:

const [listVal, setListVal] = useState<string[]>([]);

const addItem = (item: string) => {
  listVal.push(item); // 直接修改原数组
  setListVal(listVal); // 设置状态为修改后的数组
};

因为 listVal 的引用没有改变， React 可能不会检测到状态的变化，导致组件不会重新渲染。

正确的更新方式:

const [listVal, setListVal] = useState<string[]>([]);

const addItem = (item: string) => {
  setListVal([...listVal, item]); // 创建一个新数组并设置状态
};

// 如果是对象, 同理
const [objVal, setObjVal] = useState<{ key: string }>({ key: 'value' });

const updateObj = (newKey: string) => {
  setObjVal({ ...objVal, key: newKey }); // 创建一个新对象并设置状态
};
