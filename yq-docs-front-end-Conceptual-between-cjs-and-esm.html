ESM与CJS区别

修改于: 2024-07-09

ESM

// 导出方式
export const A = {};
export const B = {};
export default "Hello, world!";

CJS (即CommonJS)

// 导出方式
exports.A = {};
exports.B = {};
exports.default = "Hello, world!";

CJS导入ESM的模块

导入方式

// 本地模块
import { A, B } from "./module.mjs";

// 三方包 根据提示需要进行tsconfig相关配置
const {C} = await import("xxx");

ESM使用 __dirname

ESM默认没有 __dirname, __filename

所以使用ESM可能会遇到这个报错: __dirname is not defined in ES module scope

解决

import { dirname } from "node:path"
import { fileURLToPath } from "node:url"

export const __filename = fileURLToPath(import.meta.url);
export const __dirname = dirname(__filename);

参考: ES Modules 中的 __dirname 和 __filename

AMD

AMD是RequireJS的模块定义规范，它允许在浏览器中异步加载模块，并使用回调函数来处理模块的依赖关系。 AMD规范定义了模块的加载和依赖关系的方式，使得开发人员可以轻松地使用模块化编程。

AMD规范定义了以下三个核心概念：

  1. 模块定义：模块定义是一个函数，该函数接受一个或多个依赖项作为参数，并返回一个模块的导出对象。
  2. 模块加载：模块加载是一个异步的过程，通过调用模块定义函数，可以加载一个模块
define(['jquery', 'underscore'], function($, _) {
  // 模块代码
});

UMD

UMD是Universal Module Definition的缩写，它是一种模块定义规范，旨在解决不同JavaScript环境下的模块加载问题。 UMD规范允许模块在CommonJS、AMD和浏览器全局变量等多种环境中运行。

UMD规范定义了以下几个核心概念：

  1. 模块定义：UMD模块定义是一个函数，该函数接受一个或多个依赖项作为参数，并返回一个模块的导出对象。
  2. 模块加载：UMD模块加载是一个异步的过程，通过调用模块定义函数，可以加载一个模块。
  3. 模块导出：UMD模块导出是一个对象，该对象包含了模块的所有导出成员。
  4. 模块依赖：UMD模块依赖是一个数组，该数组包含了模块所依赖的其他模块的名称。
  5. 模块名称：UMD模块名称是一个字符串，该字符串表示模块的名称。
(function (root, factory) {
  if (typeof define === 'function' && define.amd) {
    // AMD
    define(['jquery', 'underscore'], factory);
  } else if (typeof exports === 'object') {
    // CommonJS
    module.exports = factory(require('jquery'), require('underscore'));
  } else {
    // Browser globals
  }
})
